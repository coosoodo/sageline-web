# 1. 시작하기 전에

## 1.1 이 문서에 대하여

이 문서는 **부엉이 트레이더(부엉이 주식자동매매 프로그램)** 의 공식 사용자 설명서입니다. 처음 사용하는 분부터 숙련된 사용자까지, 프로그램의 기능을 올바르게 이해하고 안전하게 활용할 수 있도록 안내합니다.

### 이 문서가 다루는 내용

설명서는 사용자가 프로그램을 처음부터 끝까지 무리 없이 운용할 수 있도록 다음 순서로 구성되어 있습니다.

| 구분 | 내용 |
|---|---|
| **설치 및 초기 설정** | 프로그램 설치와 처음 사용하기 위한 필수 준비 |
| **화면(UI) 조작 방법** | 각 화면의 구성 요소와 조작 순서 |
| **기능 활용 방법** | 자동매매 전략 설정, 조건검색, 스케줄러 등 주요 기능 사용법 |
| **결과 확인 방법** | 체결 결과, 수익률, 계좌 잔고, 로그 등 확인 방법 |

### 표기 약속

이 문서에서 사용하는 표기 방식은 다음과 같습니다.

| 표기 | 의미 |
|---|---|
| **[버튼명]** | 화면에서 클릭하는 버튼을 나타냅니다. 예: **[저장]**, **[확인]** |
| **[상위 → 하위]** | 메뉴 이동 경로를 나타냅니다. 예: **[설정 → 시스템 환경 설정]** |
| `고정폭 텍스트` | 입력값 예시나 항목명을 나타냅니다. 예: `현재가` |
| ⚠️ **주의** | 잘못 조작하면 오류가 발생하거나 의도치 않은 주문이 실행될 수 있는 항목입니다. |
| 💡 **팁** | 기능을 더 편리하게 활용할 수 있는 추가 정보입니다. |
| 📌 **참고** | 이해를 돕기 위한 보충 설명입니다. |

### 적용 버전

이 문서는 아래 버전을 기준으로 작성되었습니다.

| 항목 | 내용 |
|---|---|
| **프로그램명** | 부엉이 트레이더 (부엉이 주식자동매매 프로그램) |
| **문서 적용 버전** | 2.0.0 |
| **운영체제** | Windows 10 이상 (64비트) |
| **필수 연동** | 키움증권 REST API |

⚠️ **주의:** 이후 버전에서는 화면 구성이나 기능이 일부 변경될 수 있습니다. 사용 중인 프로그램 버전과 이 문서의 적용 버전이 다를 경우, **[도움말 → 프로그램 정보]** 에서 현재 버전을 확인한 뒤 해당 버전의 최신 문서를 참고하시기 바랍니다.

### 문서 개정 이력

| 버전 | 주요 변경 내용 |
|---|---|
| **2.0.0** | 메인 화면 구성 및 설정 화면 재편에 맞추어 설명서를 전면 재작성했습니다. |

## 1.2 부엉이 트레이더란?

**부엉이 트레이더** 는 키움증권 REST API를 기반으로, 사용자가 정해 둔 전략과 규칙에 따라 종목 선정부터 매수·매도 주문 실행까지의 과정을 자동으로 처리하는 **Windows 전용 주식 자동매매 프로그램**입니다. 복잡한 코딩 없이도, 설정한 조건에 맞추어 프로그램이 실시간으로 시장을 감시하고 대응하도록 설계되었습니다.

### 지원 증권사 및 지원 시장

| 항목 | 내용 |
|---|---|
| **지원 증권사** | 키움증권 (키움 REST API 전용) |
| **계좌 유형** | 실계좌 및 모의투자 계좌 모두 지원 |
| **지원 시장** | 코스피, 코스닥, ETF, ETN |

### 사용을 위한 필수 조건

부엉이 트레이더를 사용하려면 아래 조건이 반드시 충족되어야 합니다.

| 항목 | 내용 |
|---|---|
| **키움증권 계좌** | 국내 주식 위탁 거래 계좌 (실계좌 또는 모의투자) |
| **키움 REST API 키** | 앱키(App Key) 및 시크릿키(Secret Key) 발급 필요 |

⚠️ **주의:** 이 프로그램은 키움증권 계좌와 API 키가 없으면 사용할 수 없습니다. API 키 등록 방법은 4장을 참고하시기 바랍니다.

### 핵심 기능 요약

**① 자동매매 전략**
"어떤 종목을 언제 사고팔 것인가"를 미리 규칙으로 정의합니다. 종목은 키움 조건검색식을 활용하거나 직접 지정할 수 있으며, 단일종목·키움 조건검색식·사용자 정의의 세 가지 방식으로 전략을 만들 수 있습니다.

**② 유연한 매수·매도 조건**
단순히 종목을 고르는 데 그치지 않고, **준비 조건 → 실행 조건 → 매도 조건**으로 나누어 언제 사고팔지 세밀하게 설정할 수 있습니다. 주문 금액도 수식(예: 보유 현금의 일정 비율)으로 지정할 수 있습니다.

**③ 리스크 관리**
과도한 손실을 막기 위한 안전장치가 계좌별로 내장되어 있습니다.

| 리스크 관리 항목 | 설명 |
|---|---|
| **일일 총 매수 금액 제한** | 하루 동안 매수에 사용할 수 있는 최대 금액 |
| **일일 동일 종목 매매작업 생성 제한** | 같은 종목에 대해 하루에 만들 수 있는 매매작업 수 |
| **동일 매수주문 연속 제한** | 같은 조건의 매수 주문이 연속으로 실행되는 것을 방지 |

**④ 다양한 주문 유형**
지정가·시장가 등 상황에 맞는 주문 유형을 지원합니다.

**⑤ 자동 실행 스케줄러**
지정한 요일과 시간에 프로그램을 자동으로 시작·종료하도록 예약할 수 있습니다.

**⑥ 멀티 계좌 운용**
하나의 프로그램에서 여러 개의 키움 계좌(최대 10계좌)를 등록하여, 계좌별로 독립적인 전략과 리스크 한도를 운용할 수 있습니다.

⚠️ **투자 위험 고지:** 부엉이 트레이더는 매매 실행을 자동화하는 도구이며 **수익을 보장하지 않습니다.** 설정한 전략과 시장 상황에 따라 손실이 발생할 수 있으며, 모든 투자 결과에 대한 책임은 사용자 본인에게 있습니다. 실계좌 사용 전에는 반드시 **모의투자 계좌로 충분히 검증**하시기 바랍니다.

## 1.3 주요 기능 한눈에 보기

부엉이 트레이더의 주요 기능을 한눈에 정리하면 다음과 같습니다.

| 구분 | 주요 기능 |
|---|---|
| **자동매매 만들기** | 단일종목 / 키움 조건검색식 / 사용자 정의 — 세 가지 편집기 |
| **전략 구성 요소** | 최초매수·추가매수·매도 조건, 시스템조건식, 주문금액 계산식 |
| **리스크 관리** | 계좌별 일일 총 매수 금액·동일 종목 매매작업 생성·동일 매수주문 연속 제한 |
| **실시간 모니터링** | 자동매매 현황, 진행 중 매매작업, 주문·체결 내역, 수익률 조회 |
| **잔고·계좌 조회** | 키움잔고, 시스템잔고, 당일매매, 예수금·계좌평가 현황 |
| **종목·차트 조회** | 전체 종목정보, 주식 기본·현재·요약 정보, 분봉·일봉 차트, 기술적 지표 |
| **알림** | Discord WebHook, Telegram 알림 (계좌별 설정 지원) |
| **스케줄러** | 요일·시간 지정 자동 시작/종료 |
| **로그·시스템 정보** | 실시간 로그, 로그 관리·보관, 실시간 등록 현황, 연결 상태 |

## 1.4 화면 구성 미리보기

프로그램을 실행하면 나타나는 메인 화면은 크게 **네 개 영역**으로 구성됩니다. 각 영역의 자세한 사용법은 뒤의 "메인 화면 안내" 장에서 다룹니다.

![부엉이 트레이더 메인 화면 전체](/images/manual/main-overview.png)

| 영역 | 위치 | 역할 |
|---|---|---|
| **상단 바** | 화면 최상단 | 좌측 메뉴바 + 우측 업종지수(코스피·코스닥) 표시 |
| **좌측 패널** | 본문 왼쪽 | 계좌현황 + 자동매매 실행 스위치 |
| **자동매매 현황** | 본문 오른쪽 | 진행 중 매매작업 목록과 상세·주문·체결 현황 |
| **하단 탭 · 상태바** | 화면 아래 | 접이식 "잔고 · 체결 현황" 탭 그룹과 실시간 상태 표시 |

📌 **참고:** 부엉이 트레이더는 **다크 네이비 테마로 고정**되어 있으며, 테마를 변경하는 기능은 없습니다.

# 2. 설치 및 초기 설정

## 2.1 시스템 요구 사항

부엉이 트레이더를 정상적으로 사용하기 위해 필요한 환경입니다. 설치 전에 아래 요구 사항을 확인하시기 바랍니다.

### 운영체제

| 항목 | 내용 |
|---|---|
| **지원 OS** | Windows 10 / Windows 11 |
| **아키텍처** | 64비트 (x64) |
| **언어** | 한국어 Windows 환경 권장 |

### 하드웨어 (권장)

| 항목 | 권장 사양 |
|---|---|
| **CPU** | 4코어 이상 |
| **RAM (메모리)** | 8 GB 이상 |
| **저장 공간** | 5 GB 이상 여유 공간 |
| **화면 해상도** | 1920 × 1080 (Full HD) 이상 |
| **네트워크** | 안정적인 유선 인터넷 연결 권장 |

💡 **팁:** 실시간 시세 수신과 다수 전략 동시 실행 시 메모리 사용량이 증가합니다. 여유 있는 운용을 위해 8 GB 이상의 RAM을 권장합니다.

### 필수 연동 서비스

| 항목 | 내용 |
|---|---|
| **키움증권 계좌** | 국내 주식 위탁 거래 계좌 (실계좌 또는 모의투자) |
| **키움 REST API 키** | 앱키(App Key)·시크릿키(Secret Key) 발급 필요 |

⚠️ **투자 안정성 주의:** 자동매매 프로그램 특성상 **PC가 절전 모드로 전환되거나 화면 보호기·자동 재시작**이 동작하면 실행 중인 전략이 중단될 수 있습니다. 자동매매 실행 중에는 아래를 확인하세요.

* Windows 절전 모드 → **해제**
* 화면 보호기 → **해제**
* Windows 자동 업데이트 강제 재시작 → **활성 시간 설정**으로 재시작 방지

## 2.3 최초 실행 — 라이선스 및 투자위험 고지 동의

프로그램을 **처음 실행하면** 최종 사용자 라이선스 계약 및 투자 위험 고지 동의 창이 표시됩니다. 이 화면은 프로그램을 안전하게 사용하기 위한 중요한 안내로, **최초 1회 반드시 확인·동의**해야 합니다.

동의 창에는 프로그램의 목적과 성격, 보증 및 책임의 한계, 사용자의 의무(실계좌 사용 전 모의투자 선검증 권고), 개인정보·보안 관련 안내 등이 담겨 있습니다.

| 선택 | 동작 |
|---|---|

| **거부** | 프로그램이 종료됩니다. |

⚠️ **주의:** 동의하지 않으면 프로그램을 사용할 수 없으며 즉시 종료됩니다. 내용을 충분히 확인한 뒤 진행하시기 바랍니다.

## 2.4 키움 서버 점검 시간 자동 대기

키움증권 서버는 매일 이른 아침 짧은 점검 시간을 가집니다. 이 시간대에 프로그램을 시작하면 발급받은 접속 정보가 점검 중 무효화되어 실시간 연결이 끊길 수 있습니다.

이를 방지하기 위해, 프로그램 **시작 시각이 오전 07:50~07:55 사이인 경우** 안내 창이 표시되고 점검이 끝날 때까지 **자동으로 대기한 뒤 프로그램을 시작**합니다.

<!-- TODO 스크린샷: 키움 서버 점검 대기 창 -->

* 안내 창에는 점검 시간 안내와 함께 **남은 대기 시간**이 카운트다운으로 표시됩니다.
* 대기를 원하지 않으면 **[프로그램 종료]** 를 눌러 종료할 수 있습니다.

📌 **참고:** 이 대기 동작은 해당 시간대에 실행했을 때만 나타납니다. 그 외 시간에 실행하면 곧바로 프로그램이 시작됩니다.

# 3. 메인 화면 안내

부엉이 트레이더를 실행하면 나타나는 메인 화면의 구성과 각 영역을 읽는 방법을 안내합니다. 프로그램을 처음 사용하신다면 이 장을 먼저 읽고 화면의 전체 구조를 익혀 두시면, 이후 자동매매 설정과 모니터링이 한결 수월해집니다.

## 3.1 메인 화면 한눈에 보기

부엉이 트레이더의 메인 화면은 크게 **4개의 영역**으로 구성됩니다.

![부엉이 트레이더 메인 화면 전체 구성](/images/manual/main-overview.png)

| 영역 | 위치 | 내용 |
|---|---|---|
| **① 상단 바** | 화면 최상단 | 좌측에 **메뉴바**, 우측에 **업종지수 틱커**(KOSPI/KOSDAQ)가 한 줄에 나란히 배치됩니다. |
| **② 좌측 패널 스택** | 본문 왼쪽(폭 고정) | 위에서부터 **계좌현황** 패널과 **자동매매 실행 스위치** 패널이 세로로 쌓여 상시 표시됩니다. |
| **③ 자동매매 현황 대시보드** | 본문 오른쪽(넓은 영역) | **진행 중인 매매작업**과 선택 작업의 명세·주문·체결을 한 화면에서 모니터링·조작하는 핵심 영역입니다. |
| **④ 하단 접이식 탭 + 상태바** | 본문 아래 ~ 최하단 | **잔고 · 체결 현황** 접이식 탭 그룹과, 화면 맨 아래의 **상태바**(실시간 등록 현황·연결 상태·신호등)로 구성됩니다. |

📌 **참고:** 부엉이 트레이더는 **다크 네이비 테마로 고정**되어 있습니다. 화면 색상을 바꾸는 테마 변경 기능은 제공하지 않으며, 별도 설정 없이 항상 어두운 네이비 화면으로 표시됩니다. 야간 장시간 모니터링에도 눈의 피로가 적도록 설계된 화면입니다.

💡 **팁:** 각 영역의 목록·탭 이름 옆에 붙은 **숫자 뱃지**는 해당 영역에 표시된 항목 수를 나타냅니다. 숫자가 바뀌면 새로운 데이터가 수신된 것입니다.

## 3.2 메뉴바

화면 최상단 왼쪽의 메뉴바에서 프로그램의 모든 기능 화면을 열 수 있습니다. 메뉴는 왼쪽부터 **자동매매 · 종목정보 · 계좌정보 · 조건검색식 · 차트 데이터 · 설정 · 도움말** 순으로 배치되며, 오른쪽 끝에는 **매매제한 현황**과 **로그** 버튼이 별도로 있습니다.

📌 **참고:** 맨 앞의 **[자동매매]** 메뉴는 초록색 로봇 아이콘과 굵은 글씨로 강조되어 있습니다. 이 프로그램의 핵심 기능이 자동매매에 모여 있기 때문입니다.

### [자동매매]

자동매매 전략을 만들고 실행 현황을 조회하는, 프로그램의 핵심 기능이 모여 있습니다.

| 메뉴 항목 | 설명 |
|---|---|
| **간편 매매** | 간단한 수동 매수·매도 주문 실행 화면 |
| **자동매매 만들기 (단일종목)** | 특정 종목 하나를 대상으로 하는 전략 생성 |
| **자동매매 만들기 (키움 조건검색식)** | 키움 조건검색식에 편입되는 종목을 자동 선정하는 전략 생성 |
| **자동매매 만들기 (사용자 정의)** | 사용자가 직접 종목 목록을 지정하는 전략 생성 |
| **시스템조건식 만들기** | 매수·매도 발동 조건을 수식으로 정의하는 편집기 |
| **주문금액 계산식 만들기** | 주문 금액을 수식으로 계산하는 편집기 |
| **자동매매 현황** | 실행 중인 전략의 매매작업 현황을 별도 창으로 조회 |
| **자동매매 주문 내역** | 자동매매로 처리된 주문 이력을 별도 창으로 조회 |
| **자동매매 수익률 조회** | 자동매매 실현 수익률을 별도 창으로 조회 |

### [종목정보]

종목의 기본 정보와 시세를 조회하는 기능들입니다.

| 메뉴 항목 | 설명 |
|---|---|
| **전체 종목정보** | 상장된 전체 종목의 코드·이름 목록 조회 |
| **주식기본정보** | 종목별 기본 정보 조회 |
| **주식현재정보** | 선택 종목의 현재가·호가 조회 |
| **주식정보요약** | 종목의 핵심 지표를 요약 조회 |
| **시간외단일가** | 장 종료 후 시간외 단일가 시세 조회 |
| **변동성완화장치발동종목** | 변동성완화장치(VI)가 발동된 종목 조회 |

### [계좌정보]

키움증권 계좌의 잔고·평가 손익·예수금 현황을 상세 조회합니다.

| 메뉴 항목 | 설명 |
|---|---|
| **예수금 상세 현황** | 주문 가능 금액 등 예수금 세부 현황 |
| **계좌평가잔고내역** | 보유 종목별 평가금액 및 손익 목록 |
| **계좌평가현황** | 종목별 평가 손익 현황 요약 |

### [조건검색식]

키움증권 HTS에 등록된 조건검색식을 불러와 포착 종목을 확인합니다.

⚠️ **주의:** 이 메뉴의 두 항목은 **빨간색**으로 표시됩니다. 키움 API 사용 횟수 제한에 민감한 기능임을 알리기 위한 표시입니다.

| 메뉴 항목 | 설명 |
|---|---|
| **키움 조건검색식** | 키움 HTS에 저장된 조건검색식 목록을 불러와 조회 |
| **키움 조건검색식 (실시간)** | 조건검색식에 실시간으로 편입/이탈되는 종목을 모니터링 |

### [차트 데이터]

종목과 업종의 과거 가격 데이터, 기술적 지표를 조회합니다.

| 메뉴 항목 | 설명 |
|---|---|
| **분봉 차트 데이터** | 분 단위 가격 데이터 조회 |
| **일봉 차트 데이터** | 일별 시가·고가·저가·종가 데이터 조회 |
| **업종 일봉 차트 데이터** | 업종별 일봉 데이터 조회 |
| **기술적 지표 조회** | 피벗·볼린저밴드·일목균형표 조회 |

### [설정]

API 키 등록, 알림, 자동화 등 프로그램 운영에 필요한 설정을 합니다.

| 메뉴 항목 | 설명 |
|---|---|
| **키움증권 REST API KEY 등록** | 키움 앱키·시크릿키·계좌번호 등록 및 관리 |
| **Discord WebHook 설정** | 매매 알림을 Discord로 전송하도록 설정 |
| **Telegram 알림 설정** | 매매 알림을 Telegram으로 전송하도록 설정 |
| **시스템 환경 설정** | 당일실현손익 탭 표시, 전략상태 컬럼 표시, 계좌 자동 새로고침 등 화면 동작 옵션 제어 |
| **매매조건 처리시간 모니터** | 매매조건 처리 지연 시간을 모니터링 |
| **자동시작 스케줄러** | 자동매매 시작·종료 시간 및 요일 설정 |

📌 **참고:** **[시스템 환경 설정]**에서 켤 수 있는 화면 옵션들은 이 장 곳곳에서 안내합니다. 상세 설정 방법은 시스템 환경 설정 장을 참고하세요.

### [도움말]

프로그램 정보와 지원 관련 기능입니다.

| 메뉴 항목 | 설명 |
|---|---|
| **프로그램 정보** | 설치된 버전 정보와 라이선스 안내 표시 |
| **Log 관리** | 과거 로그 파일 조회 및 관리 |
| **후원하기** | 후원 계좌 정보 팝업 표시 |

### [매매제한 현황] · [로그] (별도 창 버튼)

메뉴바 오른쪽 끝의 두 버튼은 다른 메뉴와 달리 **각각 별도의 창**을 엽니다.

| 버튼 | 동작 |
|---|---|
| **매매제한 현황** | 계좌별 매매 제한과 키움 API 사용 제한 현황을 별도 창으로 표시합니다. 이미 열려 있으면 기존 창을 다시 활성화합니다. |
| **로그** | 프로그램 동작 로그 화면을 별도 창으로 표시합니다. 이미 열려 있으면 기존 창을 다시 활성화합니다. |

## 3.3 계좌현황 패널

메인 화면 왼쪽 위에 상시 표시되는 패널로, 등록된 계좌 목록과 선택한 계좌의 실시간 평가 요약을 보여 줍니다.

![계좌현황 패널](/images/manual/main-account-status.png)

**계좌 목록과 전환**

패널 왼쪽에는 등록된 키움 계좌 번호 목록이 표시됩니다. 계좌 번호를 클릭하면 해당 계좌로 전환되며, 오른쪽 요약 항목이 선택한 계좌 기준으로 갱신됩니다. (계좌 번호는 일부가 가려진 형태로 표시됩니다.)

**요약 항목**

선택한 계좌의 현황이 다음 항목으로 표시됩니다.

| 항목 | 설명 |
|---|---|
| **총매입금액** | 보유 종목들의 총 매입 원가 합계 |
| **총평가금액** | 현재가 기준 보유 종목들의 총 평가금액 |
| **총수익률** | 전체 보유 종목의 평균 수익률(%) |
| **총평가손익금액** | 총매입금액 대비 평가 손익 |
| **예수금** | 계좌 내 주문 가능한 현금 잔액 |

**헤더 버튼**

| 버튼 | 동작 |
|---|---|
| **새로고침** | 계좌 정보를 키움 서버에서 다시 조회합니다. |
| **설정** | 키움증권 REST API KEY 등록 화면을 엽니다. |

## 3.4 자동매매 실행 스위치

계좌현황 패널 바로 아래에 있는 패널로, 등록된 자동매매 전략을 켜고 끄는 곳입니다.

![자동매매 실행 스위치 패널](/images/manual/main-autotrading-switch.png)

**전략 목록과 실행 토글**

등록된 전략이 목록으로 표시되며, 각 전략에는 **토글 스위치**가 있습니다.

| 표시 요소 | 의미 |
|---|---|
| **토글 스위치** | 해당 전략의 자동매매 실행을 켜고(ON) 끕니다(OFF). 켜면 즉시 매매를 시작합니다. |
| **실행 중 점(●)** | 전략이 실행 중일 때만 전략 이름 옆에 초록색 점이 나타나 은은하게 명멸합니다. |
| **전략 이름** | 실행 중이면 밝고 굵게, 꺼져 있으면 흐리게 표시되어 상태를 한눈에 구분할 수 있습니다. |

여러 전략을 동시에 켤 수 있습니다. 등록된 전략이 없으면 목록은 비어 있습니다.

**자동매매 만들기 (드롭다운)**

패널 헤더 오른쪽의 톱니바퀴 버튼을 누르면 전략 만들기 드롭다운이 열립니다.

| 항목 | 설명 |
|---|---|
| **단일종목** | 특정 종목 하나를 대상으로 하는 전략을 만듭니다. |
| **조건검색식** | 키움 조건검색식으로 종목을 자동 선정하는 전략을 만듭니다. |
| **사용자 정의** | 사용자가 종목을 직접 지정하는 전략을 만듭니다. |

💡 **팁:** 같은 항목을 메뉴바 **[자동매매] → [자동매매 만들기]**에서도 열 수 있습니다.

## 3.5 자동매매 현황 대시보드

메인 화면 오른쪽 넓은 영역을 차지하는 핵심 화면으로, **진행 중인 매매작업 목록**과 선택한 작업의 **명세·주문·체결**을 한 화면에서 모니터링하고 조작합니다. 좌우 두 영역은 가운데 경계선을 드래그하여 비율을 조절할 수 있습니다.

![진행 중 매매작업 목록](/images/manual/main-active-jobs.png)

📌 **참고:** 여기서는 화면 구성과 각 버튼의 역할만 소개합니다. 매매작업을 실제로 조작하는 상세 절차는 자동매매 운영 장(6장)을 참고하세요.

### 3.5.1 좌측 — 진행 중 매매작업 목록

**헤더와 자동 새로고침**

헤더에는 "진행 중 매매작업" 제목과 건수 뱃지가 있으며, **금일 완료작업 표시** 체크박스로 오늘 완료된 작업을 목록에 함께 볼 수 있습니다. 오른쪽의 **자동 새로고침** 스위치를 켜면 목록이 **5초 주기**로 자동 갱신됩니다.

**필터**

| 필터 | 설명 |
|---|---|
| **전략명** | 특정 전략의 작업만 골라 봅니다. |
| **계좌번호** | 특정 계좌의 작업만 골라 봅니다. |
| **새로고침** | 목록을 즉시 다시 불러옵니다. |

**목록 컬럼**

| 컬럼 | 설명 |
|---|---|
| **[선택 체크박스]** | 작업을 선택합니다. 헤더 체크박스로 전체 선택도 가능합니다. |
| **계좌번호** | 작업이 속한 계좌 |
| **종목코드 / 종목명** | 대상 종목 (매도가 중지된 작업은 종목명에 "매도중지" 배지 표시) |
| **매매전략** | 작업을 만든 전략 이름 |
| **현재가** | 실시간 현재가 (▲/▼ 부호·색상) |
| **전략상태** | 각 주문 단계의 진행 상태를 원형 뱃지로 표시(옵션 컬럼, 아래 참고) |
| **생성일시** | 작업이 생성된 시각 |

행 색상으로 상태를 구분합니다. **오류**는 빨강, **완료**는 파랑 계열, **미사용**(꺼 둔 작업)은 회색·반투명으로 표시됩니다.

📌 **참고:** **전략상태** 컬럼은 기본적으로 숨겨져 있습니다. **[설정] → [시스템 환경 설정]**에서 켜면 표시되며, 각 주문 단계를 원형 뱃지로 나타냅니다(회색=대기, 주황=준비조건 통과, 초록=실행조건 통과, 테두리 파랑=매수 계열·빨강=매도). 상세는 시스템 환경 설정 장(12장)을 참고하세요.

**액션 버튼**

목록 아래(또는 목록에서 우클릭)에서 선택한 작업에 대해 다음 동작을 실행합니다.

| 버튼 | 동작 |
|---|---|
| **사용** | 선택 작업을 사용(실행) 상태로 전환 |
| **사용중지** | 선택 작업을 사용 중지 |
| **매도중지** | 선택 작업의 매도중지 ↔ 재개를 토글합니다. 중지 중에는 해당 작업의 매도 주문이 실행되지 않습니다. |
| **최초매수 미체결** | 최초매수가 아직 체결되지 않은 작업을 일괄 선택 |
| **추가매수** | 선택한 작업에 추가 매수를 실행 (작업 1개만 선택했을 때 사용) |
| **삭제** | 선택 작업을 삭제 (되돌릴 수 없는 동작, 빨강) |
| **일괄매도** | 선택 작업의 보유 수량을 즉시 전량 매도 (되돌릴 수 없는 동작, 빨강) |

⚠️ **주의:** **삭제**와 **일괄매도**는 즉시 실행되어 되돌릴 수 없습니다. 선택한 작업이 맞는지 반드시 확인하고 실행하세요. (우클릭 메뉴에는 오류 상태를 초기화하는 **에러 초기화**도 있습니다.)

**상세 정보 패널**

목록 아래에는 선택한 작업의 상세 정보를 펼쳐 볼 수 있는 접이식 패널이 있습니다(처음에는 접혀 있음). 전략명, 사용·완료·오류 여부, 주문금액, 생성일시, 사용중지 사유, 오류 메시지 등을 확인할 수 있습니다.

## 3.8 상태바 읽는 법

화면 최하단의 상태바는 프로그램의 실시간 연결 상태와 등록 현황을 보여 줍니다. 왼쪽은 **실시간 등록 현황**, 오른쪽은 **연결 상태 텍스트 · 신호등 · 후원하기**로 나뉩니다.

### 3.8.1 좌측 — 실시간 등록 현황

계좌가 하나 이상 등록되어 있으면, 상태바 왼쪽에 계좌별 실시간 등록 현황 카드가 표시됩니다.

![하단 실시간 등록 현황 바](/images/manual/statusbar-registration.png)

각 카드는 계좌 라벨과 함께 **시세 등록 수**와 **조건 등록 수**를 보여 줍니다.

| 표시 | 의미 |
|---|---|
| **시세 n/90** | 실시간 시세를 받는 종목 수 / 프로그램 상한(90). 매수 가능한 종목이 최대 90개까지임을 뜻합니다. |
| **조건 n/9** | 사용 중인 실시간 조건검색식 수 / 프로그램 상한(9) |

📌 **참고:** 키움 REST API의 실제 상한은 실시간 시세 100개, 조건검색식 10개이지만, 안정적인 처리를 위해 프로그램에서는 각각 **90개·9개**로 제한합니다.

**색상으로 보는 여유 정도**

| 색상 | 상태 |
|---|---|
| 🟢 초록 | 여유 (상한의 80% 미만) |
| 🟠 주황 | 주의 (상한의 80% 이상) |
| 🔴 빨강 | 포화 (상한 도달) |

💡 **팁:** 등록 현황 라벨 옆의 **?** 아이콘을 누르면 요청 종류별 상한과 색상 기준을 설명하는 안내 화면이 열립니다.

### 3.8.2 우측 — 연결 상태 · 신호등 · 후원하기

**연결 상태 텍스트**

키움 서버와의 연결 상태를 텍스트와 점으로 표시합니다.

| 표시 | 의미 |
|---|---|
| 🟢 **연결됨** | 모든 연결이 정상 |
| 🟠 **일부 연결** | 일부 연결만 살아 있음 — 네트워크 상태 점검 권장 |
| 🔴 **연결 끊김** | 연결이 없음 — 연결 상태 확인 필요 |

**신호등 (4종)**

연결 상태 옆에는 **사용자 아이디**와 함께 4개의 신호등이 순서대로 표시됩니다. 각 신호등에 마우스를 올리면 툴팁으로 누적 횟수를 확인할 수 있습니다.

| 신호등 | 점등 조건 | 색상 변화 |
|---|---|---|
| **① TR 요청** | 키움 API에 데이터 조회 요청이 발생할 때 | 회색 → 초록 → 1초에 걸쳐 서서히 회색 |
| **② 키움 서버 Ping 수신** | 키움 서버의 Ping을 수신할 때 | 초록 → 5초에 걸쳐 회색 |
| **③ 실시간 데이터 수신** | 실시간 시세·체결 데이터를 수신할 때 | 초록 → 0.3초 만에 빠르게 회색 |
| **④ 통신 연결 상태** | 상시 표시 (1.5초 주기 펄스) | 전체 연결=초록 / 일부 연결=주황 / 연결 없음=빨강 |

💡 **팁:** 장 중인데 **③ 실시간 데이터 수신** 신호가 전혀 깜빡이지 않거나 **④ 연결 상태**가 빨강이면, 실시간 데이터가 수신되지 않는 상태입니다. 인터넷 연결과 키움 API 등록 상태를 확인하세요.

**후원하기**

상태바 오른쪽 끝의 **후원하기** 버튼을 누르면 후원 계좌 정보 팝업이 표시됩니다. 메뉴바 **[도움말] → [후원하기]**와 동일한 기능입니다.

# 4. 키움 계좌 및 API 키 설정

부엉이 트레이더는 키움증권 REST API를 통해 주문 실행, 계좌 조회, 실시간 시세 수신 등 모든 핵심 기능을 수행합니다. 이를 위해 키움증권 REST API 포털에서 발급받은 **App Key**와 **Secret Key**를 프로그램에 등록해야 합니다.

⚠️ **주의:** App Key와 Secret Key가 하나도 등록되지 않으면 프로그램의 매매 관련 기능을 사용할 수 없습니다.

## 4.1 키움 REST API KEY 등록

메뉴 **[설정 → 키움증권 REST API KEY 등록]** 을 클릭하면 등록 화면이 열립니다.

| 버튼 | 동작 |
| --- | --- |
| **[+ 계좌 추가]** | 새 계좌 항목을 목록에 추가합니다. 계좌가 **10개 미만**일 때만 사용할 수 있습니다. |
| **[삭제]** | 선택한 계좌 항목을 목록에서 제거합니다. |
| **[초기화]** | 해당 계좌 항목에 입력한 값을 비웁니다. |

### 상세 입력 항목

좌측 목록에서 계좌를 선택한 뒤 우측에서 아래 항목을 입력합니다.

| 입력 항목 | 설명 | 예시 |
| --- | --- | --- |
| **계좌번호** | 키움증권 국내 주식 계좌번호 (하이픈 없이 숫자만) | `1234567890` |
| **App Key** | 키움 포털에서 발급받은 공개 식별자 | `PSxxxxxxxxxxxxxxxxxxxxxxxxxxx` |
| **Secret Key** | 키움 포털에서 발급받은 비공개 인증키 | `xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx` |
| **별칭** | 계좌를 알아보기 쉽게 붙이는 이름 | `주력 계좌` |
| **서버 구분** | **● 모의투자** 또는 **● 실서버** 중 선택 | `● 모의투자` |

⚠️ **주의:** **● 실서버**로 등록한 계좌에서 자동매매를 시작하면 **실제 돈으로 주문이 실행**됩니다. 처음 사용하시는 분은 반드시 **● 모의투자** 계좌로 먼저 충분히 검증하십시오.

⚠️ **주의:** Secret Key는 절대 타인에게 공개하지 마십시오. 유출 시 타인이 귀하의 계좌로 주문을 실행할 수 있습니다. 유출이 의심되면 키움 REST API 포털에서 즉시 재발급하십시오.

📌 **참고:** 계좌번호·App Key·Secret Key 중 하나라도 입력하면 나머지 항목도 함께 입력해야 합니다. 또한 서로 다른 계좌에 **같은 계좌번호를 중복** 입력할 수 없습니다.

### 연결 테스트

상세 편집 영역의 **[연결 테스트]** 버튼을 누르면, 입력한 App Key와 Secret Key로 키움 서버에 실제 접속을 시도하여 키가 유효한지 즉시 확인합니다. 결과는 성공/실패 메시지와 뱃지로 표시됩니다.

💡 **팁:** 저장 전에 **[연결 테스트]** 로 키가 올바른지 미리 확인해 두면, 저장 단계에서의 오류를 줄일 수 있습니다.

### 저장과 반영

**[저장]** 을 누르면 입력값을 검사한 뒤 유효한 계좌만 저장합니다. 최소 **1개 이상의 유효한 키**가 있어야 저장됩니다. 이때 새로 입력하거나 수정한 키만 키움 서버로 다시 검증하며, 변경하지 않은 키(별칭만 바꾼 경우 포함)는 재검증을 생략합니다.

자격 증명(App Key·Secret Key)은 **Windows 자격 증명 관리자(Credential Manager)** 에 안전하게 저장됩니다. 별도 파일로 평문 보관하지 않으므로 보안에 안전하며, PC를 재부팅해도 유지됩니다.

⚠️ **주의:** 계좌·키 변경 내용은 **프로그램을 재시작한 후에 반영**됩니다. 저장 후 안내에 따라 프로그램을 다시 시작하십시오.

## 4.2 계좌별 매매 리스크 한도

의도치 않은 과도한 주문을 막기 위해, 매매 리스크 한도를 **계좌(App Key)별로** 설정할 수 있습니다. 각 한도는 4.1의 상세 편집 영역에서 해당 계좌를 선택한 상태로 함께 입력하며, 계좌마다 서로 다른 값을 지정할 수 있습니다.

| 리스크 항목 | 설명 | 기본값 |
| --- | --- | --- |
| **일일 총 매수 금액 제한** | 해당 계좌에서 하루 동안 매수할 수 있는 총 금액 상한(원) | `100,000,000` (1억 원) |
| **일일 동일 종목 매매작업 생성 제한** | 하루 동안 같은 종목에 대해 새로 만들 수 있는 매매작업 수 | `5` 회 |
| **동일 매수주문 연속 제한** | 같은 매수주문을 연속으로 넣을 수 있는 최대 횟수 | `5` 회 |

📌 **참고:** 세 항목 모두 **계좌 단위**로 저장됩니다. 여러 계좌를 운용하는 경우, 계좌마다 자금 규모와 전략에 맞게 개별적으로 조정하십시오.

이 한도들은 **[저장]** 하는 즉시 매매제한 현황에 반영됩니다. 현재 남은 한도와 사용량은 **매매제한 현황** 화면에서 확인할 수 있습니다(6장 참조).

💡 **팁:** 리스크 한도는 예기치 못한 반복 주문이나 시스템 오작동으로 인한 손실을 줄여 주는 안전장치입니다. 실서버 운용 전에 계좌별로 적절한 값을 설정해 두시길 권장합니다.

# 5. 자동매매 전략 만들기

자동매매 전략은 부엉이 트레이더의 핵심입니다. "어떤 종목을, 언제, 얼마에, 얼마만큼 사고팔 것인가"를 미리 규칙으로 정의해 두면, 프로그램이 장중에 실시간으로 조건을 감시하며 사용자를 대신해 주문을 냅니다. 이 장에서는 전략을 만드는 세 가지 편집기와, 그 안에서 사용하는 주문 전략·시스템조건식·주문금액식·제외 설정을 순서대로 안내합니다.

## 5.1 개요

### 전략은 2계층으로 구성됩니다

부엉이 트레이더의 전략은 **매매 전략**과 **주문 전략** 두 계층으로 이루어집니다.

| 계층 | 역할 |
|---|---|
| **매매 전략** | 어떤 종목을 대상으로, 어느 계좌에서, 어떤 공통 규칙(기본 주문 금액·보유 종목수 제한·자동 실행 시간·제외 종목 등)으로 운용할지 정하는 상위 단위입니다. |
| **주문 전략** | 매매 전략 안에 여러 개를 쌓아 두는 하위 단위입니다. 각 주문 전략은 **최초매수 / 매수 / 매도** 중 하나이며, 자기만의 조건·주문유형·주문금액을 가집니다. |

즉, 하나의 매매 전략 안에 "최초매수 1건 + 추가 매수 여러 건 + 매도 여러 건" 식으로 **주문 전략을 행(行) 단위로 누적**해 나가는 방식입니다.

📌 **참고:** 이 프로그램에는 매수 탭·매도 탭처럼 화면이 나뉜 구조가 없습니다. 매수든 매도든 모두 하나의 주문 전략(행)으로 만들며, 만든 행이 화면 가운데의 **주문 전략 상세** 목록에 차곡차곡 쌓입니다.

### 세 가지 편집기 (종목 선정 방식에 따라 구분)

전략을 만드는 편집기는 **종목을 어떻게 고를 것인가**에 따라 세 종류로 나뉩니다. 상단 메뉴에서 각각 진입합니다.

| 편집기 | 진입 메뉴 | 종목 선정 방식 |
|---|---|---|
| **단일종목** | **[자동매매 만들기 (단일종목)]** | 사용자가 지정한 특정 한 종목만 매매합니다. |
| **키움 조건검색식** | **[자동매매 만들기 (키움 조건검색식)]** | 키움 HTS 조건검색식이 포착·편입하는 종목을 매매합니다. |
| **사용자 정의** | **[자동매매 만들기 (사용자 정의)]** | 특징주·호가창·외부 전략 등 프로그램 내장 방식으로 종목을 고릅니다. |

💡 **팁:** 메인 화면의 **자동매매 실행 스위치** 패널에서 우클릭하면 **[단일종목 / 조건검색식 / 사용자 정의]** 컨텍스트 메뉴로도 각 편집기를 열 수 있습니다.

📌 **참고:** 각 편집기 창은 하나만 열립니다. 이미 열려 있는 상태에서 다시 메뉴를 누르면 새 창이 뜨지 않고 기존 창이 앞으로 나옵니다.

### 세 편집기 차이점 요약

세 편집기는 화면 구성이 거의 같지만, 종목 선정 방식이 달라 일부 항목이 나타나거나 사라집니다.

| 항목 | 단일종목 | 키움 조건검색식 | 사용자 정의 |
|---|:---:|:---:|:---:|
| 종목 검색(돋보기) | O | – | – |
| 최초 매수 방법(포착/편입) | – | O | – |
| 재매수 지연시간(초) | – | O | – |
| 당일 동일종목 재매수 제한 | – | O | O |
| 당일 매수체결 없을 시 자동 완료 | – | O | O |
| 잔고 보유 종목 매수 제외 | – | O | – |
| 매매 제외 카테고리 | O | O | O |
| 매매 제외 종목(개별) | – | O | – |
| 숨김 전략 표시 | – | O | – |
| ID 표시 | – | O | O |
| 주문 전략 사용/중지 토글 | – | O | – |
| 주문 전략 실행주기 표시 | – | O | – |
| 전략 가져오기/내보내기 | O | O | O |

📌 **참고:** 표에서 "O"는 해당 편집기에 그 기능이 있다는 뜻입니다. 세 편집기 모두 공통으로 쓰는 화면 구성은 다음 절(5.2)에서, 각 편집기 전용 항목은 5.4~5.6에서 설명합니다.

## 5.2 공통 화면 구성

세 편집기는 모두 **왼쪽 전략 목록 / 오른쪽 전략 상세**의 2단 구조를 가집니다. 오른쪽 상세는 다시 위에서부터 **전략 헤더 → 주문 전략 상세 목록 → 저장/닫기 버튼** 순으로 배치됩니다.

![자동매매 만들기 (키움 조건검색식) 편집기 전체 화면](/images/manual/editor-cbs.png)

### 왼쪽: 전략 목록

지금까지 만든 매매 전략들이 목록으로 표시됩니다. 상단 툴바와 행 우클릭 메뉴로 관리합니다.

| 기능 | 설명 |
|---|---|
| **새 전략** | 빈 매매 전략을 하나 만듭니다. |
| **삭제** | 선택한 전략을 삭제합니다. |
| **위로 이동 / 아래로 이동** | 목록에서 전략 순서를 바꿉니다. |

행에서 우클릭하면 **[새로 만들기 / 삭제 / 가져오기 / 내보내기]** 메뉴가 나타납니다. 가져오기·내보내기는 5.11에서 다룹니다.

⚠️ **주의:** 설정 값에 오류가 있는 전략은 목록에서 **빨간색 글자**로 표시됩니다. 빨간 전략은 정상 동작하지 않으므로, 저장 전에 헤더와 조건을 다시 확인하시기 바랍니다.

📌 **참고:** 키움 조건검색식 편집기와 사용자 정의 편집기에서는 목록 하단의 **ID 표시** 체크로 전략마다 부여된 고유 ID를 볼 수 있습니다. 키움 조건검색식 편집기에서는 **숨김 전략 표시** 체크로 숨긴 전략까지 함께 볼 수 있습니다. (숨김 기능은 5.5에서 설명합니다.)

### 오른쪽 상단: 전략 헤더

왼쪽에서 전략을 선택하면 오른쪽에 그 전략의 설정이 나타납니다. 헤더에서 전략 전체에 적용되는 공통 항목을 정합니다.

| 항목 | 설명 |
|---|---|
| **이름** | 전략 이름입니다. (필수) |
| **계좌번호** | 이 전략으로 주문을 낼 계좌를 선택합니다. (필수) |
| **기본 주문 금액** | 주문 1건의 기준 금액입니다. 주문금액식에서 이 값을 변수로 활용할 수 있습니다. `0`원 이상 `10,000,000`원 이하로 입력합니다. |
| **보유 종목수 제한** | 이 전략이 동시에 보유할 수 있는 종목 수의 상한입니다. `0`~`90` 범위로 입력합니다. |

⚠️ **주의:** **계좌번호**와 **이름**은 반드시 지정해야 저장됩니다. 계좌번호를 고르지 않으면 어떤 계좌로 주문할지 정해지지 않아 전략이 오류로 표시됩니다.

### 오른쪽 하단: 저장 / 닫기

- **저장**: 현재 전략의 변경 내용을 저장합니다.
- **닫기**: 편집기 창을 닫습니다.

⚠️ **주의:** 헤더·주문 전략·조건 등을 바꾼 뒤에는 반드시 **[저장]** 을 눌러야 반영됩니다. 저장하지 않고 창을 닫으면 변경 내용이 사라질 수 있습니다.

## 5.3 자동 실행 시간 · 자동 완료 · 일괄 매도

전략을 언제 가동하고 언제 마무리할지는 헤더의 시간 관련 설정으로 정합니다. 이 설정들은 세 편집기에 공통으로 있습니다(일부 항목은 편집기별로 유무가 다르며, 아래에 표시했습니다).

![자동 실행 시간(A/B) 설정 패널](/images/manual/editor-auto-time.png)

### 자동 실행 시간 (A · B)

헤더 오른쪽의 접이식 **자동 실행 시간** 섹션을 펼치면, 시간대를 **시작~종료** 형태로 지정할 수 있습니다. 각 항목은 서로 다른 두 시간대 **(A)** 와 **(B)** 를 따로 켤 수 있어, 하루에 두 구간을 운용할 수 있습니다.

| 항목 | 설명 |
|---|---|
| **자동 실행 시간 (A) / (B)** | 전략 자체가 동작하는 시간대입니다. |
| **자동 매수 실행 시간 (A) / (B)** | 매수 주문을 허용하는 시간대입니다. |
| **자동 매도 실행 시간 (A) / (B)** | 매도 주문을 허용하는 시간대입니다. |

💡 **팁:** 예를 들어 매수는 오전 한 구간에만 하고 매도는 종일 열어 두고 싶다면, 자동 매수 실행 시간에만 좁은 구간을 지정하고 자동 매도 실행 시간은 넓게 지정하면 됩니다.

### 지정 일시 자동 완료

특정 날짜·시각이 되면 이 전략의 매매작업을 자동으로 마무리합니다.

- **지정 일시 자동 완료** 체크 후, **N일 후** 와 **시각** 을 지정합니다.

### 지정 일시 일괄 매도

지정한 날짜·시각에 이 전략으로 보유 중인 종목을 한꺼번에 매도합니다.

- **지정 일시 일괄 매도** 체크 후, **N일 후** 와 **시각** 을 지정합니다.
- **주문유형(일괄매도용)**: 일괄 매도 시 사용할 주문유형과 **틱** 조정을 함께 지정합니다.

### 당일 매수체결이 없는 경우 자동 완료 처리

당일 장중에 매수 체결이 한 건도 없었다면 매매작업을 자동으로 완료 처리합니다. (**키움 조건검색식**, **사용자 정의** 편집기 전용)

⚠️ **주의:** 자동 완료와 일괄 매도는 실제 주문·청산으로 이어지는 설정입니다. 날짜(N일 후)와 시각을 정확히 지정했는지 저장 전에 다시 확인하시기 바랍니다.

## 5.4 단일종목 편집기

### 종목 검색 (돋보기)

헤더에 **종목코드**와 **종목명**이 표시되고, 옆의 **돋보기 버튼**을 누르면 종목 검색 창이 열립니다. 여기서 매매 대상 종목을 골라 지정합니다.

📌 **참고:** 종목 검색(돋보기)은 단일종목 편집기에만 있습니다. 나머지 두 편집기는 조건검색식이나 내장 방식으로 종목을 자동 선정하므로 종목을 직접 지정하지 않습니다.

이후의 주문 전략·조건·주문금액 설정은 세 편집기가 동일합니다. 5.7 이하를 참고하시기 바랍니다.

## 5.5 키움 조건검색식 편집기

**키움 조건검색식 편집기**는 키움 HTS에 등록한 조건검색식이 잡아내는 종목을 자동으로 매매할 때 사용합니다. 공통 구성 외에, 조건검색 기반 운용에 필요한 항목들이 추가로 있습니다.

![자동매매 만들기 (키움 조건검색식) 편집기 전체 화면](/images/manual/editor-cbs.png)

### 최초 매수 방법 (포착 / 편입)

이 전략이 종목을 언제 매수 대상으로 삼을지를 결정합니다.

| 방법 | 설명 |
|---|---|
| **편입** | 조건검색식에 종목이 **실시간으로 새로 편입되는 순간** 매수 대상으로 삼습니다. |
| **포착** | 조건검색식을 **조회했을 때 나타나는 종목**을 매수 대상으로 삼습니다. 예를 들어 종가 무렵(예: 15:20)에 포착해 두었다가 다음 날 매도하는 종가 매매에 활용합니다. |

### 당일 동일 종목 재매수 제한

**당일 동일 종목 재매수 제한**을 켜면, 그날 이미 매수 이력이 있는 종목은 다시 최초매수하지 않습니다.

📌 **참고:** 이 제한은 최초매수에만 적용됩니다. 이미 보유 중인 종목에 대한 추가 매수는 영향을 받지 않습니다.

### 재매수 지연시간 (초)

동일 종목을 다시 매수할 때 최소로 기다릴 시간을 초 단위로 지정합니다.

📌 **참고:** 재매수 지연시간은 **당일 동일 종목 재매수 제한**을 켜지 않았을 때 사용합니다. 재매수 제한을 켜면 이 값은 사용되지 않습니다.

### 잔고 보유 종목 매수 제외

**잔고 보유 종목 매수 제외**를 켜면, 현재 계좌 잔고에 이미 들고 있는 종목은 신규 매수 대상에서 제외합니다.

📌 **참고:** 잔고 정보는 계좌 대시보드를 통해 주기적으로 갱신됩니다.

### 매매 제외 종목 (개별)

헤더 오른쪽 접이식 섹션 중 **매매 제외 종목**은 특정 종목을 개별적으로 매매에서 빼는 기능입니다. **편집** 버튼으로 제외할 종목을 종목코드·종목명 단위로 등록합니다. (자세한 조작은 5.10)

📌 **참고:** 개별 종목 제외(매매 제외 종목)는 키움 조건검색식 편집기에만 있습니다.

### 전략 숨기기 · 주문 전략 사용/중지

- 왼쪽 전략 목록에서 우클릭 시 **숨김 / 숨김 해제** 메뉴로 전략을 목록에서 감추거나 다시 표시할 수 있습니다. 숨긴 전략은 목록 하단 **숨김 전략 표시**를 켜야 보입니다.
- 오른쪽 **주문 전략 상세** 목록에서는 행을 우클릭해 **사용 / 사용 중지**를 토글할 수 있습니다. 사용 중지한 주문 전략은 흐리게 표시되며 실행되지 않습니다.

## 5.6 사용자 정의 편집기

**사용자 정의 편집기**는 조건검색식이 아니라 프로그램에 내장된 방식으로 종목을 선정하는 전략에 사용합니다. 종목 선정 방식은 다음 세 가지입니다.

| 방식 | 설명 |
|---|---|
| **특징주** | 프로그램이 특징주 기준으로 종목을 선정합니다. |
| **호가창 매매** | 호가창 흐름을 기준으로 종목을 선정합니다. |
| **외부 전략** | 외부에서 들어오는 매매 신호와 연동해 종목을 선정합니다. |

이 방식들은 준비조건·실행조건의 **사용자 정의** 유형에서 선택합니다(5.7 참고). 헤더에는 **당일 동일 종목 재매수 제한**과 **당일 매수체결 없을 시 자동 완료** 항목이 있으며, 종목 검색이나 최초 매수 방법(포착/편입)은 없습니다.

## 5.7 주문 전략 상세

주문 전략은 실제 매수·매도 규칙 한 건입니다. 오른쪽 가운데의 **주문 전략 상세** 목록에서 **추가** 또는 **수정**을 누르면 **주문전략상세 편집** 창이 열립니다. 이 창에서 매수/매도 구분, 주문 방식, 실행주기, 준비/실행조건, 주문금액을 정합니다.

### 주문 전략 상세 목록

주문 전략 목록에는 지금까지 만든 주문 전략이 행 단위로 표시됩니다.

| 컬럼 | 설명 |
|---|---|
| **매수매도** | 최초매수(빨강)·매수(주황)·매도(파랑) 배지로 표시됩니다. |
| **주문유형** | 시장가/현재가 등 이 주문의 주문유형입니다. |
| **준비조건** | 선택된 준비조건 이름입니다. |
| **실행조건** | 선택된 실행조건 이름입니다. |
| **매매금액** | 선택된 주문금액식 이름입니다. |

📌 **참고:** 키움 조건검색식 편집기에서는 여기에 **실행주기** 컬럼이 함께 표시됩니다.

목록 툴바로 **추가 / 삭제 / 수정 / 가져오기 / 위·아래 이동**을 할 수 있습니다. (가져오기는 5.11)

### 5.7.1 매수/매도 구분과 주문 방식

주문전략상세 편집 창 상단에서 이 주문 전략의 성격을 정합니다.

![주문전략상세 편집 — 최초매수](/images/manual/orderstrategy-firstbuy.png)

**매수/매도** 콤보박스에서 세 가지 중 하나를 고릅니다.

| 구분 | 설명 |
|---|---|
| **최초매수** | 보유 수량이 없는 상태에서 처음 진입하는 매수입니다. |
| **매수** | 최초매수가 이루어진 뒤 추가로 매수하는 주문입니다. |
| **매도** | 보유 종목을 매도하는 주문입니다. |

**주문 방식**은 **일반 주문**과 **복합 주문** 중 선택합니다.

- **일반 주문**: **주문유형**(시장가/현재가 등)을 고르고, 현재가 기준일 때는 **틱** 간격을 함께 지정합니다.
- **복합 주문**: 하나의 주문을 여러 주문유형·틱·비율로 나눠 냅니다. **[설정하기...]** 버튼으로 세부 설정 창을 엽니다(5.7.5).

📌 **참고:** 추가 매수는 매수/매도를 **매수**로 두고, 준비/실행조건에 물타기 조건을 넣어 구성합니다. 아래는 조건반복으로 추가 매수를 설정한 예시입니다.

![주문전략상세 편집 — 추가매수(조건반복)](/images/manual/orderstrategy-addbuy.png)

### 5.7.2 실행주기와 트리거 대상

**실행주기**는 조건이 충족될 때 이 주문 전략을 몇 번, 언제까지 실행할지를 정합니다.

| 실행주기 | 설명 |
|---|---|
| **1회** | 완료된 뒤 다시 실행하지 않습니다. |
| **매일** | 다음 날 초기화되어 매 거래일 반복 실행합니다. |
| **조건반복** | 당일 최대 3회까지 반복 실행합니다. |
| **트리거반복** | 지정한 트리거 대상 주문 전략이 완료되면 초기화되어 다시 실행합니다. |

**트리거반복**을 선택하면 **트리거 대상 선택** 버튼이 나타납니다. 이 버튼으로 어떤 주문 전략이 완료될 때 이 전략을 재실행할지 대상을 지정합니다.

![트리거 대상 주문 전략 선택 팝업](/images/manual/trigger-target-select.png)

### 5.7.3 미체결 처리 · 동적VI 재주문 · 에러 시 매매작업 중지

주문 실행 중 예외 상황을 어떻게 다룰지 정하는 옵션입니다.

| 옵션 | 설명 |
|---|---|
| **미체결 주문 처리** | 지정한 초가 지나도 체결되지 않으면 **시장가로 전환**하거나 **취소**합니다. (복합 주문일 때는 사용 불가) |
| **동적VI 거부 시 VI 해제 후 자동 재주문** | 동적 VI로 주문이 거부되면 VI 해제 시각 이후 같은 가격·수량으로 한 번 재주문합니다. 해제 시각을 알 수 없으면 재주문하지 않습니다. (복합 주문일 때는 사용 불가) |
| **에러 발생 시 현재 종목 매매작업 중지** | 이 주문에서 오류가 나면 해당 종목의 매매작업을 멈춥니다. |

⚠️ **주의:** **최초매수** 주문에서는 에러 발생 시 매매작업 중지가 항상 적용됩니다. 최초 진입 단계의 오류는 이후 추가매수·매도의 전제를 무너뜨리므로 안전하게 중지하도록 되어 있습니다.

### 5.7.4 준비조건과 실행조건

하나의 주문은 **준비조건**이 먼저 충족된 뒤 **실행조건**이 충족될 때 전송됩니다. 준비조건은 선행 대기 조건, 실행조건은 최종 실행 시점을 정하는 조건입니다.

![주문전략상세 편집 — 매도(시스템조건식)](/images/manual/orderstrategy-sell.png)

📌 **참고:** 준비조건 영역에는 우측 상단에 **사용** 토글이 있어, 끄면 준비조건 없이 실행조건만으로 주문합니다. 실행조건은 항상 사용합니다.

준비조건·실행조건은 모두 세 가지 유형 중에서 고릅니다.

| 유형 | 설명 |
|---|---|
| **시스템조건식** | 사용자가 직접 만든 수식 조건(좌항/비교/우항)을 사용합니다. **[시스템조건식 만들기]** 버튼으로 작성합니다(6.8). |
| **키움 조건검색식** | 키움 HTS 조건검색식의 신호를 조건으로 사용합니다. |
| **사용자 정의** | 특징주·호가창 매매·외부 전략을 조건으로 사용합니다. |

**키움 조건검색식**을 고르면 신호를 받는 방식을 다시 선택합니다.

| 방식 | 설명 |
|---|---|
| **편입** | 종목이 조건에 새로 편입되는 순간을 조건 충족으로 봅니다. |
| **포착** | 지정한 시각에 조건검색식을 조회해 나타나는 종목을 조건 충족으로 봅니다. 시각 선택기가 함께 표시됩니다. |
| **편입+포착** | 편입과 포착을 함께 사용합니다. **[설정...]** 버튼으로 세부 설정을 엽니다(6.7.5). |

포착 방식에 시각을 지정한 예시는 다음과 같습니다.

![주문전략상세 편집 — 최초매수, 포착+시간지정](/images/manual/orderstrategy-firstbuy-catch-time.png)

⚠️ **주의:** **편입+포착**은 최초매수 주문 전략에서는 선택할 수 없습니다.

### 5.7.5 복합 주문 세부 설정 · 편입+포착 세부 설정

**복합 주문 세부 설정** — 하나의 주문을 여러 갈래로 쪼개 냅니다.

![복합 주문 세부 설정 팝업](/images/manual/compound-order-setting.png)

- 각 행마다 **주문유형**, **틱 간격**, **비율(%)** 을 지정하고 **항목 추가**로 갈래를 늘립니다.
- 비율 합이 맞지 않으면 경고 문구가 빨간색으로 표시됩니다.

**편입+포착 세부 설정** — 포착 조회 방식을 정합니다.

![편입+포착 세부 설정 팝업](/images/manual/inclusion-search-setting.png)

| 방식 | 설명 |
|---|---|
| **1분 주기 반복 조회** | 1분마다 조건검색식을 반복 조회합니다. |
| **지정 시간에 1회 조회** | 지정한 시각(09:00~15:30 내)에 한 번만 조회합니다. |

📌 **참고:** 어느 방식을 고르든 실시간 편입 수신은 계속 유지됩니다.

### 5.7.6 주문금액 계산식

**주문금액 계산식** 항목에서 이 주문에 사용할 주문금액식을 고릅니다. 옆의 버튼으로 **주문금액식 만들기** 창을 열어 새 식을 작성할 수 있습니다(5.9).

편집을 마치면 **[확인]** 으로 주문 전략을 저장하고, **[취소]** 로 취소합니다.

### 조건 조합식 (and / or / 괄호)

조건 단위를 여러 개 만든 경우, 각 단위 사이의 논리 관계를 **조합식**으로 표현합니다.

| 조합식 예시 | 의미 |
|---|---|
| `A` | 조건 A만 충족하면 됩니다. |
| `A and B` | A와 B를 모두 충족해야 합니다. |
| `A or B` | A 또는 B 중 하나만 충족하면 됩니다. |
| `(A and B) or C` | (A와 B 모두) 또는 C가 충족되면 됩니다. |

**시스템조건식 조합** 입력란에 직접 쓰거나, **[시스템조건식 조합 간편 설정]** 버튼으로 기본 조합을 자동 입력할 수 있습니다. 논리연산자는 `and`, `or`, 괄호는 `(`, `)` 를 사용합니다.

### 라이브러리에서 가져오기

목록 툴바의 **가져오기**를 누르면 **시스템조건식 라이브러리에서 가져오기** 창이 열립니다.

![시스템조건식 라이브러리에서 가져오기 팝업](/images/manual/condexpr-library.png)

- 왼쪽 **그룹**에서 그룹을 고르고, 오른쪽 목록에서 원하는 시스템조건식을 **여러 개 선택**해 가져올 수 있습니다.
- 선택한 항목의 조건식과 조합이 미리보기로 표시됩니다.
- **[확인]** 으로 가져오고, **[선택 전체 취소]** 로 선택을 초기화합니다.

작성을 마치면 **[저장]** 을 눌러 시스템조건식을 저장합니다. 저장된 조건식은 주문전략상세 편집 창의 준비/실행조건 목록에 곧바로 나타납니다.

## 5.9 주문금액식 만들기

**주문금액식**은 주문 1건의 금액을 계산하는 식입니다. 주문전략상세 편집 창의 **[주문 금액 계산식 만들기]** 버튼으로 편집기를 엽니다.

![주문금액식 만들기 화면](/images/manual/amountexpr-editor.png)

### 용도 (매수 / 매도)

먼저 이 식을 **매수용**으로 쓸지 **매도용**으로 쓸지 **용도**를 고르고, **이름**을 입력합니다.

💡 **팁:** **[계산식을 이름으로 사용]** 을 켜면 저장 시 계산식 자체가 이름이 되어, 목록에서 식 내용을 바로 알아볼 수 있습니다.

### 좌항 / 연산자 / 우항

계산식은 한 줄로 입력합니다.

| 입력 | 설명 |
|---|---|
| **좌항** | 계산의 왼쪽 값을 목록에서 고릅니다. |
| **연산자** | 사칙연산 부호(`+`, `-`, `*`, `/`)를 고릅니다. |
| **우항** | 오른쪽 값입니다. **상수**를 직접 입력하거나, **변수** 체크 후 변수 목록에서 고릅니다. |

계산식에 사용할 수 있는 **변수**는 다음 세 가지입니다.

| 변수 | 설명 |
|---|---|
| **기본주문금액** | 전략 헤더에 지정한 기본 주문 금액 |
| **보유수량** | 현재 보유 수량 |
| **매입금액** | 현재 매입 금액 |

작성을 마치면 **[저장]** 을 눌러 저장합니다. 저장된 주문금액식은 주문전략상세 편집 창의 **주문금액 계산식** 목록에서 선택할 수 있습니다.

## 5.10 매매 종목 필터 및 제외 종목

특정 성격의 종목을 매매에서 빼고 싶을 때 사용합니다. **카테고리 단위 제외**는 세 편집기 공통, **개별 종목 제외**는 키움 조건검색식 편집기 전용입니다.

### 매매 종목 필터 (제외 카테고리 7종)

헤더의 **매매 제외 카테고리** 섹션에서 **편집** 버튼을 누르면 **매매 종목 필터** 창이 열립니다. 여기서 제외할 카테고리를 체크합니다.

![매매 종목 필터 (제외 카테고리 7종) 팝업](/images/manual/exclusion-category.png)

| 카테고리 | 예시 항목 |
|---|---|
| **거래소** | KRX, NXT |
| **감리구분** | 감리 대상 구분 |
| **시장구분** | 코스피, 코스닥, ETF, 기타 |
| **업종** | 업종 분류 |
| **회사크기분류** | 대형·중형·소형 등 규모 분류 |

### 매매 제외 종목 (개별)

키움 조건검색식 편집기에서는 헤더의 **매매 제외 종목** 섹션 **편집** 버튼으로 특정 종목을 개별 제외할 수 있습니다.

![매매 제외 종목 선택 팝업](/images/manual/exclusion-stock.png)

제외할 종목을 종목코드·종목명 단위로 등록하면, 조건검색식이 그 종목을 포착하더라도 매매하지 않습니다.

## 5.11 전략 가져오기 / 내보내기

만든 전략은 파일로 내보내 보관하거나 다른 곳에서 다시 불러올 수 있습니다.

### 매매 전략 내보내기 / 가져오기 (JSON)

왼쪽 전략 목록에서 행을 우클릭하면 **내보내기**와 **가져오기** 메뉴가 있습니다.

- **내보내기**: 선택한 전략을 `JSON` 파일로 저장합니다. 기본 파일명은 `{전략이름}.json` 입니다.
- **가져오기**: `JSON` 파일을 선택해 전략을 불러옵니다. 불러온 전략은 현재 선택한 계좌로 편입됩니다.

📌 **참고:** 가져오기·내보내기는 세 편집기 모두에서 사용할 수 있습니다.

### 주문 전략 상세 복사 (다른 매매 전략에서)

주문 전략 상세 목록 툴바의 **가져오기**를 누르면, 같은 종목 선정 방식의 **다른 매매 전략**을 골라 그 전략의 주문 전략들을 현재 전략으로 복사해 올 수 있습니다.

![전략(주문전략상세) 가져오기 대상 선택 팝업](/images/manual/editor-import-strategy.png)

⚠️ **주의:** 자기 자신과 같은 매매 전략을 선택하면 복사할 수 없다는 경고가 표시됩니다. 서로 다른 전략을 선택하시기 바랍니다.

### 시스템조건식 라이브러리

시스템조건식은 라이브러리에서 여러 개를 골라 가져올 수 있습니다. 자세한 내용은 5.8의 **라이브러리에서 가져오기**를 참고하시기 바랍니다.

# 6. 자동매매 실행 및 모니터링

매매전략을 만들었다면 이제 실제로 실행하고, 어떻게 매매가 진행되는지 지켜볼 차례입니다. 이 장에서는 전략을 켜고 끄는 방법, 진행 중인 매매작업을 실시간으로 살펴보는 방법, 그리고 주문·수익률·매매제한을 조회하는 여러 화면을 차례로 안내합니다.

## 6.1 자동매매 시작·중지

자동매매는 **매매전략 단위**로 켜고 끕니다. 메인 화면 좌측의 **[자동매매 실행 스위치]** 패널(3.4에서 소개)에서 전략별 토글 스위치를 눌러 즉시 ON/OFF 할 수 있으며, 편집기를 열 필요가 없습니다.

### 6.1.1 전략을 켤 때 — 실행 전 순차 검증

전략 스위치를 **ON**으로 전환하면 프로그램이 곧바로 매매를 시작하지 않고, 먼저 아래 세 가지를 **순서대로** 검사합니다. 하나라도 통과하지 못하면 스위치는 자동으로 **OFF로 원복**되고 안내 메시지가 표시됩니다.

| 순서 | 검증 항목 | 통과하지 못하면 |
|---|---|---|
| 1 | **실시간 주식 체결 등록 한도** — 실시간 체결 데이터를 받을 수 있는 종목 수 한도를 넘지 않는지 확인 | "실시간 주식 체결 데이터 서비스에 등록할 수 있는 종목코드의 최대 갯수를 초과했습니다…" 메시지 후 원복 |
| 2 | **조건검색식 등록 한도** — 실시간 조건검색식 등록 개수 한도를 넘지 않는지 확인 | "조건검색 서비스에 등록할 수 있는 종목코드의 최대 갯수를 초과했습니다…" 메시지 후 원복 |
| 3 | **포착 조건검색식 충돌** — 이 전략의 포착 조건검색식이 이미 실행 중인 다른 전략에서 **편입** 조건검색식으로 쓰이고 있지 않은지 확인 | "포착 조건검색식 : '…' 가 이미 실행중인 자동매매전략에서 편입 조건검색식으로 사용되고 있습니다…" 메시지 후 원복 |

세 검증을 모두 통과하면 전략이 실행 상태로 저장되고, 실시간 체결 데이터와 편입 조건검색식이 등록되면서 자동매매 감시가 시작됩니다.

📌 **참고:** 위 1·2번의 한도는 **키움 REST API의 계좌별 한도**를 따릅니다. 프로그램이 사용할 수 있는 상한(실시간 체결 종목 수, 실시간 조건검색식 개수)은 화면 하단 등록 현황 바와 **[매매제한 현황]** 창(6.7)에서 실시간으로 확인할 수 있습니다. 현재 사용량이 한도에 가까울수록 새 전략을 켜기 어려워지므로, 한도가 넘칠 때는 사용하지 않는 전략을 먼저 중지하십시오.

💡 **팁:** 스위치 목록에는 **사용중**이면서 **오류가 없는** 전략만, 그리고 현재 선택한 계좌에 연결된 전략만 표시됩니다. 켜려던 전략이 보이지 않는다면 계좌 선택, 사용 여부, 오류 상태를 먼저 확인하십시오. 또한 등록된 실제 계좌가 아니면 해당 전략은 오류 상태로 전환되며 "…계좌번호가 존재하지 않습니다." 메시지가 표시됩니다.

### 6.1.2 전략을 끌 때 — 장중 중지 경고

이미 실행 중인 전략을 장중에 **OFF**로 전환하면 다음 확인 경고가 표시됩니다.

> "장중에 실행을 중지하면 처리되지 못한 매매 데이터가 발생할 수 있습니다.
> 이로인해 예기치 못한 오류가 발생할 수 있습니다.
> 계속 진행하시겠습니까 ?"

- **[예]** 를 선택하면 즉시 전략이 중지됩니다. 이때 이미 접수된 주문은 자동으로 취소되지 않으며, 이후 체결 이벤트에 대한 후속 처리가 누락될 수 있습니다.
- **[아니오]** 를 선택하면 스위치가 원래 ON 상태로 돌아가고 매매가 계속됩니다.

⚠️ **주의:** 갑작스러운 장중 중지는 미체결 주문이나 잔고 불일치를 유발할 수 있습니다. 가급적 보유 종목과 미체결 주문이 없는 상태에서, 또는 장 마감 후에 중지하시기를 권장합니다.

## 6.2 메인 현황 대시보드에서 모니터링

전략을 실행하면 조건에 포착된 종목마다 **매매작업(Job)**이 생성됩니다. 이 매매작업들이 어떻게 진행되는지는 메인 화면 우측에 상시 표시되는 **자동매매 현황 대시보드**에서 실시간으로 확인합니다. 이 대시보드가 v2.0.0의 주된 모니터링 수단입니다.

![메인 화면 진행 중 매매작업 대시보드](/images/manual/main-active-jobs.png)

화면은 크게 두 부분으로 나뉩니다. **좌측**은 진행 중인 매매작업 목록과 필터·액션 버튼, **우측**은 선택한 작업의 **명세 / 주문 / 체결**을 위에서 아래로 3단으로 보여주는 드릴다운 영역입니다.

### 6.2.1 진행 중 매매작업 목록

좌측 상단 헤더에는 진행 중 작업 수를 나타내는 카운트 칩, **[금일 완료작업 표시]** 체크박스, 그리고 **[자동 새로고침]** 토글이 있습니다. 자동 새로고침을 켜면 목록이 **5초 주기**로 자동 갱신됩니다.

바로 아래 **필터 패널**에서 **전략명**과 **계좌번호**를 골라 원하는 작업만 추려 볼 수 있고, **[새로고침]** 버튼으로 즉시 다시 불러올 수 있습니다.

목록(표)에는 다음 컬럼이 표시됩니다.

| 컬럼 | 설명 |
|---|---|
| (체크박스) | 작업 선택. 헤더의 체크박스로 전체 선택 |
| 계좌번호 | 작업이 속한 계좌 |
| 종목코드 | 매매 대상 종목 코드 |
| 종목명 | 종목명. 매도가 중지된 작업에는 **매도중지** 빨간 배지가 붙음 |
| 매매전략 | 작업을 생성한 전략 이름 |
| 현재가 | 실시간 현재가. 상승 ▲ / 하락 ▼ 부호와 색상 표시 |
| 전략상태 | 명세별 진행 상태를 색 원형 배지로 표시(기본 숨김) |
| 생성일시 | 작업이 생성된 일시 |

💡 **팁:** 행 색상으로 상태를 빠르게 구분할 수 있습니다. **오류** 작업은 빨간 글씨, **완료** 작업은 파란색 계열, **미사용** 작업은 회색으로 흐리게 표시됩니다.

### 6.2.2 작업 액션 (하단 버튼 · 우클릭 메뉴)

목록에서 작업을 선택한 뒤 하단 버튼이나 우클릭 메뉴로 아래 동작을 실행합니다.

| 액션 | 동작 |
|---|---|
| **사용** | 선택한 작업을 사용 상태로 전환(매매 재개) |
| **사용중지** | 선택한 작업을 사용 중지(매매 중단) |
| **매도중지** | 선택한 작업의 매도중지 상태를 토글. 중지 시 매도주문이 실행되지 않음 |
| **최초매수 미체결** | 최초매수가 아직 체결되지 않은 작업을 일괄 선택 |
| **추가매수** | 작업 **1개만 선택**했을 때 사용. 금액 입력 창이 뜨고, 입력한 금액만큼 **시장가**로 추가 매수 |
| **삭제** ⚠️ | 선택한 작업을 삭제 |
| **일괄매도** ⚠️ | 선택한 작업의 보유수량을 **즉시 매도** |
| 에러 초기화 | 작업의 오류 상태를 해제 |

⚠️ **주의:** **삭제**와 **일괄매도**는 되돌릴 수 없는 파괴적 동작입니다. 특히 **일괄매도**는 선택한 모든 작업의 보유 물량을 시장가로 즉시 처분하므로, 실행 전에 선택된 작업과 종목을 반드시 다시 확인하십시오.

### 6.2.3 우측 드릴다운 — 명세 / 주문 / 체결

좌측에서 작업을 선택하면 우측 3단에 그 작업의 세부 내역이 연동되어 나타납니다.

**우측 상단 — 매매작업 상세(명세)**
선택한 작업에 적용된 주문 전략 명세(최초매수·추가매수·매도 등 단계별 설정)가 나열됩니다. 준비·실행·주문구분·거래구분·주문금액과 함께 **준비 조건**·**실행 조건**의 통과/대기 여부, 사용여부, 완료여부, 사용중지이유, 오류 및 오류 메시지가 표시됩니다. 헤더 우측의 범례로 행 배경색의 의미(준비통과=주황, 실행통과=초록, 완료=파랑, 오류=빨강, 미사용=회색)를 확인할 수 있습니다.

명세 행에서 **우클릭**하면 다음을 실행할 수 있습니다.

- **사용 시작** / **사용 중지**
- **실행조건 대기중으로 변경** — 실행 조건을 다시 대기 상태로 되돌림
- **준비조건 대기중으로 변경** — 준비 조건을 다시 대기 상태로 되돌림

**우측 중단 — 주문**
선택한 작업에서 발생한 주문 목록입니다. 종목코드·종목명·주문구분·거래구분·생성일·주문시간·주문수량·주문가격·체결수량·미체결수량·주문번호·처리상태를 표시하며, 취소된 주문은 회색으로 나타납니다. 우클릭으로 주문을 삭제할 수 있습니다.

**우측 하단 — 체결**
선택한 작업의 실제 체결 내역입니다. 종목코드·종목명·주문구분·체결수량·체결가격·체결시간·체결번호를 표시합니다. 하나의 주문이 여러 건으로 나뉘어 체결될 수 있습니다.

📌 **참고:** 좌측 목록 아래에는 접이식 **선택한 매매작업 상세 정보** 영역이 있어, 펼치면 전략명·사용여부·완료여부·오류여부·주문금액·생성일시·사용중지사유·오류메시지를 한눈에 볼 수 있습니다.

## 6.3 매매작업 상세창

메인 대시보드에서 행을 **더블클릭**하면 더 자세한 별도 상세창이 열립니다. 더블클릭 위치에 따라 두 종류의 창이 있습니다.

⚠️ **주의:** 이 더블클릭 상세창들은 **메인 임베드 대시보드(6.2)에서만** 열립니다. 뒤에서 설명하는 별도 **[자동매매 현황]** 창(6.4)에는 더블클릭 상세창이 없습니다.

### 6.3.1 매매작업 상세 — 포지션형

**진행 중 매매작업** 목록(좌측)에서 작업 행을 **더블클릭**하면 포지션·평가 중심의 상세창이 열립니다. 계좌 잔고를 기준으로 보유 현황과 손익을 한 화면에 정리해 보여주며, **현재가는 실시간으로 반영**됩니다.

![매매작업 상세창(포지션형)](/images/manual/job-detail-window.png)

상단 포지션/평가 박스에는 다음 항목이 표시됩니다.

| 항목 | 설명 |
|---|---|
| 보유수량 | 현재 보유 중인 수량 |
| 현재가 | 실시간 현재가 |
| 매입단가 | 평균 매입 단가 |
| 매입금액 | 매수에 들어간 총 금액 |
| 평가금액 | 현재가 × 보유수량 |
| 손익률 | 손익 비율(%), 부호에 따라 색상 표시 |
| 평가손익 | 평가금액 − 매입금액, 부호에 따라 색상 표시 |

그 아래 상세 필드 박스에는 전략명·사용여부·완료여부·오류여부·주문금액·생성일시·사용중지사유·오류메시지가 표시됩니다.

### 6.3.2 매매작업 상세 — 명세형(2×2 카드)

**매매작업 상세(명세)** 목록(우측 상단)에서 명세 행을 **더블클릭**하면, 넓은 명세 표를 읽기 쉽게 **2×2 카드 4개**로 정리한 상세창이 열립니다.

| 카드 | 담긴 내용 |
|---|---|
| **기본** (좌상) | 주문구분 · 거래구분 · 실행주기 · 주문금액 |
| **상태** (우상) | 사용여부 · 완료여부 · 오류 · 오류 메시지 · 사용중지이유 · 생성일시 |
| **준비 조건** (좌하) | 유형 · 조건식 · 조합 · 사용자정의 · **통과 여부**(통과/대기) |

💡 **팁:** 준비 조건과 실행 조건이 **모두 통과** 상태여야 해당 주문이 실제로 발송됩니다. 주문이 나가지 않는다면 이 두 카드의 통과 여부를 먼저 확인하십시오.

## 6.4 자동매매 현황 (별도 창)

메인 대시보드 외에, 자동매매 전체 계층을 한 창에서 넓게 보고 싶을 때는 별도 **[자동매매 현황]** 창을 사용합니다. 상단 메뉴 **[자동매매 → 자동매매 현황]** 을 클릭하면 열립니다.

![자동매매 현황(별도 창)](/images/manual/autotrading-status.png)

화면은 좌에서 우로 **5개 영역**이 나란히 배치되며, 왼쪽 항목을 선택하면 오른쪽이 연동되는 드릴다운 방식입니다.

| 영역 | 내용 |
|---|---|
| ① 매매전략 | 등록된 전략 목록(계좌번호·전략명). 하단에 기본 주문 금액·매수시간 A/B·매도시간 A/B·사용 여부·실행 여부·기타 설명 등 상세 표시 |
| ② 작업중 매매작업 | 선택 전략의 작업 목록(계좌번호·종목코드·종목명·생성일시). **[금일 완료작업 표시]** 체크박스 제공, 우클릭으로 삭제. 하단에 작업 상세 표시 |
| ③ 매매작업 상세(명세) | 선택 작업의 명세 목록(준비·실행·주문구분·거래구분·주문금액·준비 조건·실행 조건·사용여부·사용중지이유·오류·오류 메세지·생성일시) |
| ④ 주문 현황 | 선택 명세의 주문 목록(종목·주문구분·거래구분·주문시간·주문/체결/미체결수량·주문가격·주문번호·처리상태) |
| ⑤ 체결 현황 | 선택 주문의 체결 내역(종목·주문구분·체결수량·체결가격·체결시간·체결번호) |

우측 상단 **[새로고침]** 버튼으로 전체 데이터를 다시 불러오며, 각 영역 경계는 드래그하여 크기를 조절할 수 있습니다.

📌 **참고:** 이 창은 전체 계층을 넓게 훑어보기에 좋지만, 앞서 설명한 더블클릭 상세창(6.3)은 열리지 않습니다. 포지션·평가나 명세 2×2 카드를 보려면 메인 대시보드를 이용하십시오.

## 6.5 자동매매 주문 내역

과거의 주문과 체결 내역을 전략·기간별로 조회하려면 상단 메뉴 **[자동매매 → 자동매매 주문 내역]** 을 클릭합니다.

![자동매매 주문 내역(별도 창)](/images/manual/autotrading-order-history.png)

### 조회 방법

1. 상단에서 **전략**을 선택합니다.
2. 기간을 지정합니다. **1일 · 2일 · 1주일 · 1개월 · 1년** 빠른 버튼을 쓰거나, 시작일과 종료일을 직접 골라도 됩니다.
3. **[조회]** 버튼을 클릭합니다. 하단 상태바에 **총 N건**이 표시됩니다.

### 주요 컬럼

| 주문가격 · 주문수량 | 주문 시 가격과 수량 |
| 체결수량 · 체결평균가 · 체결금액 | 실제 체결 결과 |

## 6.6 자동매매 수익률 조회

매매작업 단위로 수익률을 요약·집계해서 보려면 상단 메뉴 **[자동매매 → 자동매매 수익률 조회]** 를 클릭합니다. 창 제목은 **자동매매 매매작업별 수익률 조회**입니다.

![자동매매 수익률 조회(별도 창)](/images/manual/autotrading-profit.png)

### 조회 방법

전략을 선택하고 기간(**1일 · 2일 · 1주일 · 1개월 · 1년** 또는 직접 지정)을 정한 뒤 **[조회]** 를 누르면, 상태바에 조회된 작업 수가 표시됩니다.

### 합계 요약 바

화면 상단에는 조회된 전체 작업의 합계가 한 줄로 요약됩니다.

| 지표 | 설명 |
|---|---|
| 매입금액 합계 | 매수에 들어간 총 금액 |
| 매도금액 합계 | 매도로 회수한 총 금액 |
| 실현손익 합계 | 매도금액 − 매입금액 |
| 수익률 | 실현손익 기준 평균 수익률 |
| 실현손익(순) 합계 | 수수료·세금을 반영한 실현손익 |
| 수익률(순) | 실현손익(순) 기준 평균 수익률 |

📌 **참고:** 지표 이름에 붙는 **"순"** 은 **수수료와 세금을 반영한 뒤**의 값이라는 뜻입니다. 실제 손에 남는 금액에 가까운 것은 "순" 지표입니다.

### 작업별 목록과 상세 주문

합계 아래에는 매매작업별 요약 표가 있습니다(작업ID·전략명·종목명·매입금액·매도금액·실현손익·수익률·실현손익(순)·수익률(순)·첫 주문·마지막 주문). 작업을 선택하면 하단에 그 작업의 **상세 주문 내역**(주문일시·구분·체결수량·매입가·매도가·결과·메시지)이 표시됩니다.

작업별 요약 표는 **우클릭 → [Excel 내보내기]** 로 저장할 수 있습니다.

## 6.7 매매제한 현황

자동매매는 키움 REST API와 프로그램 자체의 여러 제한 안에서 동작합니다. 각 계좌가 현재 어느 한도를 얼마나 쓰고 있는지 게이지로 실시간 확인하려면, 상단 메뉴 우측의 **[매매제한 현황]** 을 클릭해 별도 창을 엽니다.

![매매제한 현황(별도 창)](/images/manual/trading-limit-status.png)

창에는 계좌별 카드가 배치되며, 각 카드에 다음 **6종 제한 항목**이 게이지로 표시됩니다. 사용량이 한도에 가까워지면 게이지 색이 붉게 변합니다.

| 제한 항목 | 의미 |
|---|---|
| **총 매수금액 제한** | 사용한 매수금액 / 최대 매수금액 한도(원) |
| **실시간 주식 체결 데이터 요청 제한** | 구독 중인 종목 수 / 최대(REST API 제한). 구독 종목 수는 매매작업 개수(중복 종목 제외) 기준. **[종목보기]** 버튼으로 요청 종목 목록(코드·종목명)을 볼 수 있음 |
| **조건검색식 요청 제한** | 실시간 조건검색식 개수 / 최대(REST API 제한) |
| **일일 동일 종목 매매작업 생성 제한** | 하루에 같은 종목으로 만들 수 있는 매매작업 최대 횟수. 초과한 종목은 코드·횟수로 표시되며(초과 없으면 "제한 초과 종목 없음"), **자정에 초기화** |
| **연속 동일 매수주문 제한** | 같은 종목에 대한 연속 동일 매수주문 최대 횟수. 다른 주문이 발생하면 초기화 |
| **보유 종목수 제한 (전략별)** | 전략별 보유 종목 수 / 최대. 한도에 도달하면 해당 전략의 신규 매수가 제한됨 |

각 항목 옆의 **?** 도움말을 누르면 그 제한이 무엇을 뜻하는지 요약 설명을 볼 수 있습니다. 초과된 종목은 강조 색으로 표시됩니다.

💡 **팁:** 6.1에서 전략을 켤 때 검증하는 **실시간 체결 등록 한도**와 **조건검색식 등록 한도**가 바로 이 창의 2·3번 항목입니다. 전략이 켜지지 않거나 새 매매작업이 만들어지지 않을 때, 이 창에서 어떤 한도가 가득 찼는지 먼저 확인하면 원인을 빠르게 찾을 수 있습니다.

## 7.1 간편 매매 열기

메뉴에서 **[자동매매 → 간편 매매]**를 선택하면 "간편 주문" 창이 열립니다. 창은 세로로 긴 소형 창으로, 화면 위쪽에는 **계좌번호** 선택 콤보가, 그 아래에는 **매수 / 매도 / 취소** 세 개의 탭이 있습니다.

- 창을 열면 사용할 수 있는 계좌 중 **모의투자 계좌가 우선 자동으로 선택**됩니다. 모의투자 계좌가 없으면 실제 계좌가 선택됩니다.
- 주문을 넣기 전에 화면 위쪽 계좌번호가 원하는 계좌인지 반드시 확인하십시오.

💡 **팁:** 처음 창을 열면 수량은 `1`주, 가격은 `50,000`원이 기본값으로 채워져 있습니다. 종목을 검색하면 현재가가 자동으로 반영되므로, 값을 그대로 두지 말고 종목을 먼저 지정하는 것이 좋습니다.

## 7.2 매수 · 매도

**매수 탭**과 **매도 탭**은 입력 항목이 같습니다. 아래 항목을 채운 뒤 각 탭의 **[매수]** 또는 **[매도]** 버튼을 누릅니다.

| 입력 항목 | 설명 |
|---|---|
| **종목** | 직접 입력하지 않고 **[검색]** 버튼을 눌러 종목 선택 창에서 고릅니다. 종목을 고르면 이름이 자동으로 채워집니다. |
| **종류** | 주문 종류를 콤보에서 선택합니다. |
| **수량** | 매수 또는 매도할 주식 수를 입력합니다. |
| **가격** | 주문 가격을 입력합니다. 종목을 검색하면 그 종목의 현재가가 자동으로 채워지며, 호가 단위에 맞게 자동 보정됩니다. |

- 종목을 검색하면 현재가를 조회해 **가격**란에 자동으로 넣어 줍니다. 필요하면 값을 직접 고쳐 입력할 수 있습니다.
- 매수 탭 버튼과 매도 탭 버튼은 색으로 구분되어, 매수는 붉은색, 매도는 파란색 계열로 표시됩니다.

## 7.3 주문 취소

이미 넣은 주문 중 아직 체결되지 않은 주문은 **취소 탭**에서 취소합니다.

| 입력 항목 | 설명 |
|---|---|
| **종목** | 취소할 주문의 종목이 표시됩니다(직접 입력 불가). |
| **번호** | **[미체결]** 버튼을 눌러 미체결 주문 목록에서 취소할 주문을 고르면 주문 번호가 채워집니다. |
| 수량 · 가격 | 선택한 주문의 정보가 자동으로 표시되며, 취소 화면에서는 수정할 수 없습니다. |

원하는 미체결 주문을 고른 뒤 **[취소]** 버튼을 누르면 취소 주문이 전송됩니다.

## 7.4 주문 시 유의사항

⚠️ **주의:** 매수·매도·취소 모두 버튼을 누르면 곧바로 실행되지 않고, 먼저 **확인 창**이 한 번 표시됩니다. 내용을 확인한 뒤 확인을 눌러야 실제 주문이 전송됩니다. 실제 자금이 오가는 주문이므로 계좌·종목·수량·가격을 꼭 다시 살펴보십시오.

⚠️ **주의:** **키움 서버 점검 시간(오전 07:50 ~ 07:55)** 에는 주문이 차단되며, 이 시간대에 주문을 시도하면 경고가 표시됩니다. 점검이 끝난 뒤 다시 시도하십시오.

📌 **참고:** 간편 매매는 종목 하나를 그때그때 직접 주문하는 용도입니다. 조건에 따라 자동으로 매매하려면 자동매매 기능을 사용하십시오.

|---|---|

# 9. 자동 시작 스케줄러

자동 시작 스케줄러는 부엉이 트레이더를 지정한 요일과 시간에 **자동으로 시작하고 종료**하도록 예약하는 기능입니다. 매일 직접 프로그램을 켜고 끌 필요 없이, 장 시작 시간에 맞춰 프로그램이 자동으로 실행되도록 Windows에 예약해 둘 수 있습니다.

## 9.1 스케줄러 화면 열기

상단 메뉴에서 **[설정 → 자동시작 스케줄러]** 를 선택하면 스케줄 설정 창이 열립니다.

![자동시작 스케줄러 화면](/images/manual/scheduler.png)

⚠️ **주의:** 스케줄을 저장하려면 **반드시 관리자 권한으로 프로그램을 실행**해야 합니다. 일반 권한으로 실행 중이면 저장 시 "관리자 권한으로 실행해주세요." 라는 안내가 표시되고 저장되지 않습니다. 관리자 권한으로 실행하려면 **내 문서\SageLine\StartSageStockPilot.vbs** 파일을 실행하십시오. UAC(사용자 계정 컨트롤) 창이 나타나며 관리자 권한으로 프로그램이 시작됩니다.

| **요일 선택** | 스케줄을 적용할 요일(월~일)을 개별로 선택합니다. |
| **저장 / 취소** | 설정을 저장하거나 창을 닫습니다. |

📌 **참고:** 화면 안내 문구 기준으로 **자동시작 가능 시간대는 07:30 ~ 23:00** 입니다. 이는 키움 REST API의 사용 가능 시간을 고려한 범위이므로, 시작·종료 시간은 이 범위 안에서 설정하시는 것을 권장합니다.

## 9.3 자동 시작 · 종료 설정 방법

시작 스케줄과 종료 스케줄은 **하나의 화면에서 함께** 설정하고 한 번에 저장합니다.

**① 자동시작 활성화**

- **[자동시작 활성화]** 스위치를 **ON** 으로 설정합니다.

**② 시작 시간 입력**

- **[시작 시간]** 에 프로그램을 실행할 시각을 입력합니다. (예: 장 준비를 위해 `08:50`)
- 07:30 이후로 설정하는 것을 권장합니다.

**③ 종료 시간 입력**

- **[종료 시간]** 에 프로그램을 종료할 시각을 입력합니다. (예: 장 마감 후 `16:00`)
- 23:00 이전으로 설정하는 것을 권장합니다.

⚠️ **주의:** 종료 시각이 되면 진행 중인 자동 매매가 있더라도 프로그램이 종료됩니다. 사용 중인 모든 매매 전략이 마무리되는 시각 이후로 종료 시간을 설정하십시오.

**④ 요일 선택**

- 스케줄을 실행할 요일을 체크합니다. 평일 자동 매매라면 **월·화·수·목·금** 을 선택합니다.
- 최소 **1개 이상**의 요일을 선택해야 저장이 가능합니다.

**⑤ 저장**

- **[저장]** 버튼을 클릭합니다. 시작 시간, 종료 시간, 요일이 모두 설정되어야 저장 버튼이 활성화됩니다.
- 저장하면 프로그램을 자동으로 실행·종료하는 예약 작업이 Windows 작업 스케줄러에 등록됩니다. 시작 시각에는 **내 문서\SageLine\StartSageStockPilot.vbs** 와 같은 자동 실행 스크립트가 사용됩니다.

## 9.4 스케줄 확인 및 해제

**현재 설정 확인**

스케줄러 창을 다시 열면 **이전에 저장한 설정값(활성화 여부, 시작·종료 시간, 선택 요일)이 그대로 표시**됩니다. 현재 어떤 스케줄이 예약되어 있는지 한눈에 확인할 수 있습니다.

**스케줄 수정**

값을 변경한 뒤 **[저장]** 을 다시 클릭하면 됩니다. 기존 예약 작업은 자동으로 갱신되므로 별도의 삭제 작업은 필요하지 않습니다.

**스케줄 해제**

**[자동시작 활성화]** 스위치를 **OFF** 로 바꾸고 **[저장]** 을 클릭하면, 등록되어 있던 시작·종료 예약 작업이 자동으로 삭제됩니다.

## 9.5 자주 발생하는 문제

| 증상 | 원인 및 해결 방법 |
| --- | --- |
| 저장 시 "관리자 권한으로 실행해주세요." 표시 | 일반 권한으로 실행 중입니다. **내 문서\SageLine\StartSageStockPilot.vbs** 로 다시 실행한 뒤 시도하십시오. |
| 저장 버튼이 눌리지 않음 | 시작 시간·종료 시간·요일 중 입력되지 않은 항목이 있습니다. 모두 설정한 뒤 다시 시도하십시오. |
| 설정한 시각에 프로그램이 실행되지 않음 | **[자동시작 활성화]** 가 OFF 상태이거나 예약 작업이 등록되지 않았을 수 있습니다. 스케줄러 창에서 활성화 여부와 저장 상태를 확인하십시오. |

# 10. 로그 및 진단

부엉이 트레이더는 동작 중 발생하는 모든 이벤트를 기록하고, 그 기록을 확인·관리할 수 있는 도구를 제공합니다. 이 장에서는 실시간 로그를 확인하는 **로그 대시보드**, 로그 파일을 관리하는 **로그 관리**, 그리고 매매조건 판단 속도를 점검하는 **매매조건 처리시간 모니터**를 안내합니다.

## 10.1 로그 대시보드

로그 대시보드는 실시간으로 발생하는 사용자 로그와 오류 로그를 필터링해 확인하는 **별도 창**입니다.

**화면 열기**

상단 메뉴 우측의 **[로그]** 버튼을 클릭하면 로그 대시보드 창이 열립니다. 프로그램 시작 시 함께 표시되기도 합니다.

![로그 대시보드 화면](/images/manual/log-dashboard.png)

**화면 구성**

| 구성 요소 | 설명 |
| --- | --- |
| **일반 로그** | 매매작업 생성, 주문·체결, 매수/매도 성공·실패, 시스템 메시지 등 프로그램 동작 내역이 실시간으로 쌓입니다. |
| **오류 로그** | 프로그램에서 발생한 오류만 별도로 모아 표시합니다. |
| **사용자 UI 조작 로그 기록** | 사용자의 화면 조작 내역까지 로그로 남길지 여부를 켜고 끄는 옵션입니다. |

**일반 로그 필터**

일반 로그는 아래 세 가지 기준으로 원하는 내용만 걸러 볼 수 있습니다.

| 필터 | 설명 |
| --- | --- |
| **유형** | 매매작업 생성, 주문체결, 매수성공, 매도성공, 매수실패, 매도실패, System 등 로그 종류로 필터링합니다. |
| **계좌** | 특정 계좌에서 발생한 로그만 표시합니다. |
| **매매전략** | 특정 매매 전략과 관련된 로그만 표시합니다. |

📌 **참고:** 일반 로그와 오류 로그는 각각 최대 10,000건까지 유지됩니다. **[사용자 UI 조작 로그 기록]** 을 켜면 화면 조작 내역까지 남길 수 있어 문제 재현·분석에 도움이 되지만, 로그 양이 늘어납니다.

## 10.2 로그 관리

로그 관리 창에서는 저장된 로그 파일을 열람·삭제하거나 지원팀 서버로 전송할 수 있고, 로그 보관 기간을 설정할 수 있습니다.

**화면 열기**

상단 메뉴에서 **[도움말 → Log 관리]** 를 선택하면 로그 관리 창이 열립니다.

![로그 관리 화면](/images/manual/log-manager.png)

**로그 파일 저장 위치**

로그 파일은 다음 경로에 자동 저장됩니다.

`내 문서\SageLine\SageStockPilot\logs`

**주요 기능**

| 기능 | 설명 |
| --- | --- |
| **파일 목록** | 로그 폴더에 있는 모든 로그 파일을 표시합니다. |
| **보관 일수** | 로그 보관 기간을 **3 / 5 / 7 / 14 / 30 / 90 / 180 / 365일** 중에서 선택합니다. 프로그램 시작 시 오래된 로그는 자동으로 압축되고, 보관 기간이 지난 로그는 자동 삭제됩니다. |
| **메모장으로 열기** | 선택한 로그 파일 1개를 Windows 메모장으로 엽니다. 압축(.zip)된 로그는 내부 파일을 임시로 꺼내 열어 줍니다. |
| **삭제** | 선택한 로그 파일을 삭제합니다. (여러 개 선택 가능) |
| **서버로 전송** | 선택한 로그 파일을 지원팀 서버로 업로드합니다. 오류 분석 문의 시 활용합니다. |

⚠️ **주의:** **서버로 전송** 은 **파일당 최대 2GB** 까지만 가능합니다. 2GB를 초과하는 로그 파일은 업로드되지 않습니다. 전송 중에는 대기 화면이 표시되며, 완료될 때까지 프로그램을 강제 종료하지 마십시오.

💡 **팁:** 메모장으로 로그를 연 뒤 `Ctrl + F` 로 종목 코드나 오류 메시지 등 특정 키워드를 빠르게 검색할 수 있습니다.

# 11. 알림 설정

부엉이 트레이더는 매매 중 발생하는 주요 소식을 **Discord** 와 **Telegram** 으로 실시간 전송할 수 있습니다. PC 앞에 없더라도 스마트폰으로 체결·오류·경고 등을 즉시 확인할 수 있습니다. 두 채널은 서로 독립적이므로 하나만 사용하거나 둘 다 함께 사용할 수 있습니다.

두 채널 모두 **알림 옵션**으로 전송 범위를 조절할 수 있습니다.

| 알림 옵션 | 전송 범위 |
| --- | --- |
| **일반** | 주문·체결 알림을 포함한 일반 알림을 모두 전송 |
| **중요 알림** | 체결·오류·경고 등 주요 알림만 전송 |

## 11.1 Discord 알림

### 11.1.1 WebHook URL 발급

WebHook을 사용하려면 **서버 관리 권한**이 있는 Discord 계정과 알림을 받을 채널이 필요합니다.

- Discord를 실행하고 알림을 받을 **서버**와 **텍스트 채널**을 준비합니다. (전용 채널을 새로 만들어도 됩니다.)
- 채널 이름에서 마우스 오른쪽 클릭 → **[채널 편집]** → **[연동]** 탭 → **[웹후크 만들기]** 를 클릭합니다.
- 웹후크 이름을 입력하고 **[웹후크 URL 복사]** 버튼을 눌러 URL을 복사한 뒤 저장합니다.

발급된 URL은 `https://discord.com/api/webhooks/...` 형식입니다.

⚠️ **주의:** WebHook URL은 해당 채널에 누구나 메시지를 보낼 수 있는 열쇠와 같습니다. 타인과 공유하거나 공개된 곳에 게시하지 마십시오.

### 11.1.2 부엉이 트레이더에서 설정하기

상단 메뉴에서 **[설정 → Discord WebHook 설정]** 을 클릭하면 설정 창이 열립니다.

![Discord WebHook 설정 화면](/images/manual/discord-setting.png)

| 항목 | 설명 |
| --- | --- |
| **WebHook URL** | 앞서 복사한 URL을 붙여넣습니다. |
| **사용 여부** | 알림 전송을 켜고 끕니다. |
| **알림 옵션** | **일반** 또는 **중요 알림** 을 선택합니다. |
| **테스트 전송** | 설정한 채널로 확인 메시지를 보내 연결을 점검합니다. |

- 값을 입력한 뒤 **[테스트 전송]** 버튼을 누르면 Discord 채널로 확인 메시지가 전송됩니다. 전송에 실패하면 HTTP 코드가 함께 표시되므로 URL을 다시 확인하십시오.
- 설정을 저장하면 완료됩니다.

## 11.2 Telegram 알림

### 11.2.1 Bot Token과 Chat ID 준비

- Telegram에서 **@BotFather** 를 검색해 대화를 시작하고 `/newbot` 명령으로 봇을 생성합니다. 생성이 완료되면 `123456:ABC-DEF...` 형식의 **Bot Token** 이 발급됩니다.
- 알림을 받을 본인 계정 또는 그룹의 **Chat ID** 를 확인합니다. `@userinfobot` 에게 말을 걸면 본인의 Chat ID를 알려줍니다.

### 11.2.2 부엉이 트레이더에서 설정하기

상단 메뉴에서 **[설정 → Telegram 알림 설정]** 을 클릭하면 설정 창이 열립니다.

![Telegram 알림 설정 화면](/images/manual/telegram-setting.png)

| 항목 | 설명 |
| --- | --- |
| **Bot Token** | BotFather에서 발급받은 토큰을 입력합니다. |
| **기본 Chat ID** | 계좌별 Chat ID가 지정되지 않은 경우 사용되는 기본 수신처입니다. |
| **사용 여부** | 알림 전송을 켜고 끕니다. |
| **알림 옵션** | **일반** 또는 **중요 알림** 을 선택합니다. |
| **계좌별 Chat ID** | 계좌마다 다른 채널로 알림을 받고 싶을 때 계좌별로 Chat ID를 지정합니다. 계좌번호와 서버 구분(실서버/모의투자)이 함께 표시됩니다. |
| **테스트 전송** | 기본 Chat ID 또는 각 계좌 Chat ID로 확인 메시지를 보냅니다. |

📌 **참고:** 계좌별 Chat ID를 지정하면 그 계좌의 알림은 해당 채널로 전송되고, 지정하지 않은 계좌의 알림은 **기본 Chat ID로 전송**됩니다. 계좌마다 알림 채널을 나누고 싶을 때 유용합니다.

### 11.2.3 테스트

- **[테스트 전송]** 으로 기본 Chat ID 연결을 확인합니다.
- 계좌별 목록의 각 계좌 **[전송]** 버튼으로 해당 계좌의 Chat ID로 개별 테스트 메시지를 보낼 수 있습니다.

## 11.3 자주 묻는 질문 (FAQ)

**Q. 알림 전송에 실패하면 매매에 영향이 있나요?**
없습니다. 알림은 매매 동작과 독립적으로 처리되므로, 전송이 실패해도 자동 매매에는 영향을 주지 않습니다.

**Q. 스마트폰에서도 받을 수 있나요?**
네. 스마트폰에 Discord 또는 Telegram 앱을 설치하고 해당 채널의 알림을 켜 두면 푸시 알림으로 수신할 수 있습니다.

**Q. Discord와 Telegram을 동시에 쓸 수 있나요?**
네. 두 채널은 독립적으로 동작하므로 함께 설정하면 양쪽 모두로 알림이 전송됩니다.

# 12. 시스템 환경 설정

프로그램의 화면 표시, 로그 기록, 계좌현황 자동 새로고침과 관련한 환경 옵션을 설정합니다. 메뉴 **[설정 → 시스템 환경 설정]** 을 클릭하면 설정 화면이 열립니다.

![시스템 환경 설정 화면](/images/manual/system-setting.png)

아래 옵션을 원하는 대로 선택한 뒤 **[저장]** 하면 **즉시 반영**됩니다.

## 12.1 표시 및 로그 옵션

| 옵션 | 설명 |
| --- | --- |
| **메인화면에 '당일실현손익' 탭 표시** | 메인 화면에 당일 실현손익을 확인하는 탭을 추가로 표시합니다. |
| **매매전략 조건 판단 상세 로그 기록** | 매매전략의 조건 판단 과정을 상세하게 로그로 남깁니다. **해제하면** 로그 기록에 따른 오버헤드가 줄어듭니다. |
| **자동매매 현황에 '전략상태' 컬럼 표시** | 자동매매 현황 화면에 각 작업의 전략상태를 보여 주는 컬럼을 추가합니다. |

💡 **팁:** 매매 성능을 최우선으로 두고 운용하실 때는 **매매전략 조건 판단 상세 로그 기록**을 해제하면 처리 부담을 조금 더 낮출 수 있습니다.

## 12.2 계좌현황 자동 새로고침

**계좌현황 자동 새로고침 사용**을 체크하면, 메인 화면의 계좌현황이 일정 주기마다 자동으로 갱신됩니다. 사용을 켜면 **새로고침 주기**를 조절할 수 있습니다.

| 항목 | 범위 | 기본값 |
| --- | --- | --- |
| **새로고침 주기** | `3` ~ `60` 초 | `15` 초 |

⚠️ **주의:** 새로고침 주기는 **10초 이상**을 권장합니다. 주기를 너무 짧게 설정하면 키움 서버에 부하가 커질 수 있으므로 주의하십시오.

## 13.3 자주 묻는 질문 (FAQ)

### 키움 REST API KEY 등록

**Q. 키가 저장되지 않아요.**
키 등록 시 다음 조건을 확인하십시오.

- 최소 **1개 이상의 유효한 키** 가 입력되어 있어야 저장됩니다.
- 한 계좌에서 **계좌번호·App Key·Secret Key** 중 하나라도 입력하면 나머지 항목도 모두 입력해야 합니다.
- **계좌번호는 중복**될 수 없습니다.

**Q. 등록한 키가 올바른지 미리 확인할 수 있나요?**
키 등록 화면의 **[연결 테스트]** 버튼을 누르면 해당 계좌의 키로 키움 서버 연결을 확인하고, 성공·실패 결과를 알려 줍니다.

**Q. 키를 변경했는데 반영되지 않아요.**
키(자격증명) 변경 내용은 **프로그램을 다시 시작한 후 반영**됩니다. 저장 후 프로그램을 재시작하십시오.

### 자동 매매 시작

**Q. 자동 매매를 시작하려면 무엇이 준비되어야 하나요?**
**유효한 키움 REST API KEY 등록**이 완료되어 있어야 합니다. 키가 하나도 등록되어 있지 않으면 정상적으로 매매를 시작할 수 없습니다.

**Q. 지정한 시간에 프로그램이 자동으로 시작되지 않아요.**
자동 시작 스케줄러는 **관리자 권한으로 실행 중일 때만** 저장·동작합니다. **내 문서\SageLine\StartSageStockPilot.vbs** 로 프로그램을 다시 실행한 뒤 스케줄을 저장하고, **[자동시작 활성화]** 가 켜져 있는지 확인하십시오. (자세한 내용은 9장 참고)
